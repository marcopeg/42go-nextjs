# Dynamic Pages Layout [hx68]

**TaskID**: HX68
**Status**: draft

the _DynamicPages_ feature is currently under `app/(public)/[...slug]` and inside `(public)` there is the layout definition that implements the `PublicLayout` component.

I propose to move the `[...slug]` outside the `(public)` and use the `PublicLayout` as a simple component there.

This way, one of the possible configuration of a public page would be to define the layout to use:

```ts
const config = {
    default: {
        public: {
            pages: {
                HomePage: {
                    layout: CustomComponent
                    items: []
                }
            }
        }
    }
}
```
