# Manual Test - SA84 Rename Public Themes to App Themes

## Preconditions

1. Start the app with the project-supported command, usually `make app.start`.
2. Open browser devtools before loading each page.
3. Keep the Network tab filtered to CSS.
4. Keep the Console tab visible for stylesheet warnings.

## Default Fallback

1. Open a host or route that resolves to the default app.
2. Confirm the document head links `/app-themes/_default/style.css`.
3. Confirm there is no request to `/themes/default.css`.
4. Confirm there is no failed CSS request.
5. Confirm the page still renders with the base token styling.

## App-Specific Theme

1. Open each configured app host that has an app theme file:
   - `app1`
   - `app2`
   - `calendar`
   - `quicklist`
   - `lingocafe`
2. For each app, confirm the document head links `/app-themes/_default/style.css`.
3. For each app, confirm the document head links `/app-themes/<app-id>/style.css`.
4. Confirm `/app-themes/<app-id>/style.css` returns `200`.
5. Confirm there is no request to `/themes/<app-id>.css`.
6. Confirm the app-specific visual token override is visible on the page.

## Missing App Theme Fallback

1. Temporarily create or use a configured app that has no `public/app-themes/<app-id>/style.css`.
2. Open that app.
3. Confirm the document head links `/app-themes/_default/style.css`.
4. Confirm the document head does not link `/app-themes/<app-id>/style.css`.
5. Confirm the Network tab has no failed app-theme CSS request.
6. Confirm the Console has no strict MIME or missing stylesheet warning.

## Regression Checks

1. Run `rg "public/themes|/themes/" src contents docs/articles docs/memory-bank`.
2. Confirm no live source or documentation references mention the legacy path.
3. Run `npm run qa`.
4. Confirm the production build completes.
5. Start the built app if the normal workflow supports it.
6. Repeat one default-app check and one app-specific-theme check against the built app.
