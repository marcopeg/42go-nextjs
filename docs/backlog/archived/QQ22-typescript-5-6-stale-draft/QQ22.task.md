# TypeScript 5 → 6

**Order**: ④ — after WP85, before UT41
**Workflow**: `arefine` then `aplan` — ts5to6 output is unpredictable, may surface many type errors

## Elevator's Pitch

Upgrade TypeScript from 5.x to 6.x. TypeScript 6 changes several default compiler options (strict, module, target, types). Use the official `ts5to6` migration tool to adjust tsconfig, then fix any new type errors.

## Business Gain

Access to latest TypeScript features and improved type checking. Stays on a supported major version.

## Current State

- `typescript` at `^5` in package.json, installed 5.8.3
- Latest: 6.0.3
- Project uses strict TypeScript with Next.js
- Supports multiple apps (default, quicklist, notes, calendar) in single codebase
- All apps compile and pass type checking under current TS5 setup

## Desired State

- TypeScript 6.x installed
- `tsconfig.json` updated for new defaults via ts5to6
- All new type errors fixed (comprehensive refactoring if needed)
- All configured apps (default, quicklist, notes, calendar) verify without type errors
- `npm run qa` passes with zero type errors on all apps

## Definition of Success

- TypeScript 6.x installed and working
- All type errors surfaced by TS6 are fixed (no errors suppressed)
- All configured apps (default, quicklist, notes, calendar) compile cleanly
- `npm run qa` passes with zero errors
- No runtime regressions

## Additional Context

Steps:

1. Run `npx ts5to6@latest` — adjusts tsconfig defaults automatically
2. Change `"typescript": "^5"` → `"typescript": "^6"`
3. `npm install`
4. Add `"types": ["node"]` to tsconfig.json explicitly (new requirement in TS6)
5. Fix any new type errors surfaced by changed defaults
6. `npm run qa`

Key TS6 changes:

- `strict` default changes
- `module` default changes
- `target` default changes
- `types` no longer includes `@types/*` automatically — must be explicit
- The `ts5to6` tool handles most of the tsconfig migration
