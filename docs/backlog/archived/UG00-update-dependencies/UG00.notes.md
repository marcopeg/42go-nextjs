### 3. `typescript` — 5.8.3 → 6.0.3

| Field | Value |
|---|---|
| **Risk** | 🔴 High |
| **Current** | 5.8.3 (installed) / `^5` (package.json range) |
| **Target** | 6.0.3 |

**Breaking Changes:**

TS 6.0 changes **default compiler options** significantly. If your `tsconfig.json` doesn't explicitly set these, behavior will change:

| Option | Old Default | New Default | Impact |
|---|---|---|---|
| `strict` | `false` | `true` | All strict checks enabled by default |
| `module` | `commonjs` | `esnext` | Module output format changes |
| `target` | `es5` | `es2025` | Emit target changes |
| `rootDir` | computed | `.` | May affect relative output paths |
| `types` | auto-discover | `[]` | **Must add `"types": ["node"]` explicitly** |

**Deprecations** (warnings now, errors in TS 7.0):

- `--moduleResolution node` → use `node16` or `bundler`
- `--baseUrl` for non-path-mapping → not needed with modern resolution
- `--esModuleInterop false` → must be true or omitted
- `import ... assert` → use `import ... with`
- `target: es5` → minimum `es2015`

**Migration Steps:**

1. Run `npx ts5to6` (official automated migration tool) to update `tsconfig.json`
2. Add `"types": ["node"]` to `compilerOptions` if not already present
3. Verify `moduleResolution` is `bundler` (Next.js default) — not `node`
4. Add `"ignoreDeprecations": "6.0"` temporarily if deprecation warnings are overwhelming
5. Bump `package.json` to `"typescript": "^6"`
6. Run `npm run qa`

**Recommendation:** Upgrade. This project's `tsconfig.json` is managed by Next.js which already uses `"moduleResolution": "bundler"`. The biggest risk is the `types: []` default — must ensure `"types": ["node"]` is set. The `ts5to6` tool handles most changes automatically. TS 6.0 is a bridge to TS 7.0 (Go rewrite) so staying on 5.x will become painful quickly.

2. Remove any `/* eslint-env */` comments from source files
3. Bump to `"eslint": "^10.0.0"`
4. Run `npm run qa` — fix any new `eslint:recommended` rule violations
5. Migration guide: https://eslint.org/docs/latest/use/migrate-to-10.0.0

**Recommendation:** Upgrade. This project already uses flat config (`eslint.config.mjs`), so the biggest breaking change (eslintrc removal) is irrelevant. The main risk is new rules from `eslint:recommended` flagging existing code.
