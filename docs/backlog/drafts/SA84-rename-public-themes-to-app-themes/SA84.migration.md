# Migration - SA84 Rename Public Themes to App Themes

## Goal

Move app-specific CSS token overrides from `public/themes` to `public/app-themes` and make stylesheet injection resolver-driven.

## Legacy Layout

```text
public/themes/<app-id>.css
```

Legacy runtime link:

```html
<link rel="stylesheet" href="/themes/<app-id>.css">
```

## New Layout

```text
public/app-themes/_default.css
public/app-themes/<app-id>.css
```

New runtime links:

```html
<link rel="stylesheet" href="/app-themes/_default.css">
<link rel="stylesheet" href="/app-themes/<app-id>.css">
```

The app-specific link is emitted only when the app-specific file exists in the generated registry.

## Steps

1. Create `public/app-themes/_default.css`.
2. Move every `public/themes/*.css` file into `public/app-themes/`.
3. Delete the legacy `public/themes` folder once all files are moved.
4. Add a shared resolver such as `resolveAppThemeStylesheet(appId)`.
5. Add a build-time registry of existing files under `public/app-themes`.
6. Update `src/42go/config/InjectAppID.tsx` to use the resolver output.
7. Audit `src/42go/config/InjectAppName.tsx`; update it or remove it if obsolete.
8. Replace all live `public/themes` and `/themes/` documentation references with `public/app-themes` and `/app-themes/`.
9. Run `rg "public/themes|/themes/"` and confirm no live references remain except historical completed backlog logs.
10. Run `npm run qa`.

## App File Mapping

```text
public/themes/app1.css       -> public/app-themes/app1.css
public/themes/app2.css       -> public/app-themes/app2.css
public/themes/calendar.css   -> public/app-themes/calendar.css
public/themes/quicklist.css  -> public/app-themes/quicklist.css
public/themes/lingocafe.css  -> public/app-themes/lingocafe.css
```

## No Compatibility Layer

Do not keep a `/themes` route, symlink, rewrite, or fallback. This project is early. The old path should die cleanly.
