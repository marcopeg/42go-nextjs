# Split Bookshelf Into Reading and Alphabetical Catalog Sections

## Elevator's Pitch

Update the LingoCafe `/books` shelf so books the user is currently reading appear first in their own section, and sort both the reading section and the remaining catalog alphabetically by title.

## Business Gain

Returning readers can jump straight back into active books, while the rest of the shelf becomes easier to scan because the catalog order is predictable. Chuck Norris does not hunt through a shelf for the book he already started. The book steps forward.

## Current State

The current `/books` page renders one grid from `data.books` and keeps the API order returned by `/api/lingocafe/books`. In `src/app/api/(lingocafe)/lingocafe/_lib/reader.ts`, the catalog is currently ordered by `level` and then `title`. The UI already knows which books are actively being read because `readingAction.kind === "resume"` is used for the reading ribbon and direct resume behavior, but those books are still mixed into the main shelf.

## Desired State

The bookshelf should prioritize active reading. When one or more books are in `resume` state, the page should show a first section for currently reading books and a second section for the rest of the available catalog. Within each section, books should be ordered alphabetically by title. The currently reading section should contain only reading books, and the lower catalog/bookshelf section should contain the remaining books without duplicates.

## Definition of Success

A user opening `/books` can immediately see active reading books at the top, can still browse the full remaining shelf below, and finds both sections sorted alphabetically by title rather than by level.

## Additional Context

The existing reading-state groundwork already exists: `AEO` added the visible reading indicator and direct-open resume behavior for books whose `readingAction.kind` is `resume`. The current page implementation lives in `src/app/(app)/(lingocafe)/books/page.tsx`, which renders the returned book array directly. The operator also suggested making the experience more explicit with two sections: a reading section first and a catalog/bookshelf section below.

## Assumptions

- A book counts as "currently reading" when `readingAction.kind` is `resume` and the book has a valid resume href.
- Alphabetical sorting should be based on book title and should not depend on reading level.
- If no books are currently being read, the page can fall back to a single general catalog section instead of showing an empty reading section.

## Constraints

- Preserve the existing language-filtered catalog behavior from `/api/lingocafe/books`.
- Preserve the current click behavior: reading books still resume directly, non-reading books still open the details page.
- Do not regress the existing shelf/card visual design work.
- Keep the task scoped to bookshelf ordering and section structure, not to a broader books-page redesign.

## Acceptance Criteria

- [ ] Books that are currently being read are surfaced before non-reading books on `/books`.
- [ ] The page uses `readingAction.kind === "resume"` as the reading-state signal unless implementation proves a stronger existing signal is already available.
- [ ] When at least one book is currently being read, the UI shows a dedicated first section for those reading books.
- [ ] The remaining available books are shown in a separate catalog/bookshelf section below the reading section.
- [ ] No book appears in both sections.
- [ ] Books inside the reading section are sorted alphabetically by title.
- [ ] Books inside the remaining catalog/bookshelf section are sorted alphabetically by title.
- [ ] If no books are currently being read, the page does not show a misleading empty reading section.
- [ ] Existing reading-book resume navigation still works.
- [ ] Existing non-reading book navigation to book details still works.
- [ ] Language filtering, onboarding behavior, empty-state handling, and current card rendering are not regressed.

## Dos

- Do reuse the reading-state data that already exists in the books payload.
- Do make the section split obvious enough that active reading books feel intentionally prioritized.
- Do keep alphabetical ordering deterministic for users.
- Do keep implementation compatible with the current `/books` page and reader data flow.

## Don'ts

- Don't keep ordering primarily driven by reading level for this bookshelf surface.
- Don't duplicate books across sections.
- Don't change the established meaning of reading-book clicks versus non-reading-book clicks.
- Don't turn this task into a new filtering system or a redesign of unrelated books-page controls.
