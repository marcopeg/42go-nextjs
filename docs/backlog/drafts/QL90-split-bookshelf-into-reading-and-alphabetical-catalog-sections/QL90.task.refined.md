# Split Bookshelf Into Reading and Alphabetical Catalog Sections

## Elevator's Pitch

Update the LingoCafe `/books` shelf so currently reading books are grouped first in their own section, while the remaining catalog stays below in a separate section. Sort both sections alphabetically by title.

## Business Gain

Returning readers can jump straight back into active books, and the rest of the shelf becomes easier to scan because the order is stable and predictable. Chuck Norris does not dig through a shelf for the book he already started. The book reports for duty.

## Current State

The current `/books` page renders one grid from `data.books` and keeps the API order returned by `/api/lingocafe/books`.

In `src/app/api/(lingocafe)/lingocafe/_lib/reader.ts`, the books query is currently ordered by `level` and then `title`.

The UI already knows which books are actively being read because `readingAction.kind === "resume"` drives the reading ribbon and direct resume behavior introduced by `AEO`, but those books are still mixed into the main shelf instead of being grouped at the top.

## Desired State

The bookshelf should make active reading obvious.

When at least one book is in reading state, the page should render:

- a first `Currently Reading` section containing only books whose `readingAction.kind` is `resume`
- a second `Catalog` section containing the remaining available books

Books in both sections should be ordered alphabetically by title.

No book should appear in both sections.

If the user is not currently reading any book, the page should fall back to the normal catalog view without rendering an empty `Currently Reading` section.

Existing click behavior should stay intact:

- reading books still resume directly into the reader
- non-reading books still open the book details page

## Definition of Success

A user opening `/books` immediately sees in-progress books at the top, sees the remaining catalog below, and can scan both sections in alphabetical order by title instead of the current level-first order.

## Additional Context

The groundwork for reading-state awareness already exists:

- `AEO` added the visible reading indicator and direct-open resume behavior for books whose `readingAction.kind` is `resume`
- `PK51` established the real language-filtered `/books` catalog
- the current page implementation in `src/app/(app)/(lingocafe)/books/page.tsx` renders the returned book array directly

The operator explicitly refined the intent:

- put currently reading books first
- sort by title alphabetically
- prefer two sections, with reading first and the rest of the bookshelf below

This task is about bookshelf ordering and grouping. It is not a broader redesign of the books page.

## Assumptions

- A book counts as "currently reading" when `readingAction.kind === "resume"` and the book has a valid resume href.
- Alphabetical sorting should compare book titles and should not be influenced by reading level.
- The existing reading-state data in the books payload is sufficient, so the task should not need a new persistence signal.
- If no books are currently being read, showing only the general catalog section is the correct fallback.

## Constraints

- Preserve the existing language-filtered catalog behavior from `/api/lingocafe/books`.
- Preserve the current click behavior: reading books still resume directly, non-reading books still open the details page.
- Do not regress onboarding behavior, empty states, or current book-card rendering.
- Keep the task scoped to bookshelf ordering and section structure, not a broader books-page redesign.
- Keep the implementation aligned with the authenticated app-page conventions already used by `/books`.

## Acceptance Criteria

- [ ] Books that are currently being read are surfaced before non-reading books on `/books`.
- [ ] Reading state is determined from `readingAction.kind === "resume"` unless implementation proves a stronger existing signal is already available.
- [ ] When at least one book is currently being read, the UI shows a dedicated `Currently Reading` section first.
- [ ] The remaining available books are shown in a separate `Catalog` section below the reading section.
- [ ] No book appears in both sections.
- [ ] Books inside the `Currently Reading` section are sorted alphabetically by title.
- [ ] Books inside the `Catalog` section are sorted alphabetically by title.
- [ ] The implementation no longer presents the bookshelf primarily in level-first order on this surface.
- [ ] If no books are currently being read, the page does not show an empty or misleading `Currently Reading` section.
- [ ] Existing reading-book resume navigation still works.
- [ ] Existing non-reading book navigation to book details still works.
- [ ] Language filtering, onboarding behavior, empty-state handling, and current card rendering are not regressed.

## Dos

- Do reuse the reading-state data that already exists in the books payload.
- Do make the section split visually obvious enough that active reading books feel intentionally prioritized.
- Do keep alphabetical ordering deterministic for users.
- Do keep implementation compatible with the current `/books` page and reader data flow.

## Don'ts

- Don't keep the bookshelf primarily ordered by reading level on this surface.
- Don't duplicate books across sections.
- Don't change the established meaning of reading-book clicks versus non-reading-book clicks.
- Don't turn this task into a new filtering system or a redesign of unrelated books-page controls.
