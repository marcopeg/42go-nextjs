# Replace Books Filter Text With Round Language Flag

## Elevator's Pitch

On the LingoCafe `/books` page, replace the current subtitle text such as `Catalog filtered for Swedish.` with a round language flag shown on the `Books` header row, aligned to the right edge and vertically centered.

## Business Gain

The books header becomes more visual, compact, and language-focused. Users can understand the active reading language at a glance without burning vertical space on a sentence. Chuck Norris does not read a status paragraph when a flag can finish the job.

## Current State

The current books page renders `Books` as the title and uses the subtitle line to show text like `Catalog filtered for Swedish.` when `targetLang` is available.

This comes from `src/app/(app)/(lingocafe)/books/page.tsx`, where `targetLabel` is converted into the subtitle string.

The provided screenshot shows that text under the `Books` title.

## Desired State

Instead of showing the `Catalog filtered for ...` sentence, the books header should show the active learning language as a round flag icon on the same header row as `Books`, aligned to the right edge and vertically centered.

The title `Books` should remain.

The textual filter sentence should be removed when the flag is shown.

The flag stays lightweight in this task:

- it is primarily a visual indicator
- tapping it routes the user to `/profile`
- it does not open a popup or inline language switcher

If there is no active learning language, the page should show no replacement flag in that slot. That state means the user is still completing the profile.

The visual treatment should match the screenshot intent: cleaner header, compact flag, and no extra explanatory text.

## Definition of Success

A user landing on `/books` can immediately identify the active reading language from a round flag badge in the header row, the old `Catalog filtered for ...` subtitle text is gone, and tapping the flag routes to `/profile` without introducing a broader switcher flow.

- The active flag should represent the current `targetLang` / learning language, not `ownLang`.
- This request is primarily a header presentation change and does not require popup or panel behavior.
- The existing books data/profile load already provides enough information to determine the active language for rendering the flag.
- Routing the flag to `/profile` is sufficient for this task and should not be expanded into a direct inline switcher flow.

## Constraints

- Do not broaden this task into the full `BL01` language-and-level switcher flow.

- Keep the implementation aligned with the authenticated app-page conventions already used by `/books`.
