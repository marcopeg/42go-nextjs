# Reading Scroll Indicator on Book Page

## Elevator's Pitch

Add a subtle reading progress indicator to the book page reading view so users can always see how far they are in long content.

## Business Gain

Improves reading UX and orientation in long pages, which can increase completion rates and perceived polish.

## Current State

The book page reading view has no visible reading progress indicator while scrolling.

## Desired State

A 2px bright blue progress bar is displayed over the header bottom border and remains visible while scrolling. The bar grows from left to right based on reading progress. At the end of the scrollable content, it touches both left and right borders.

## Definition of Success

In the book reading view, scrolling down visibly grows a subtle 2px bright blue bar across the header bottom border until full width at the end of content.

## Additional Context

Focus is explicitly limited to the book page reading view.

## Assumptions

- Progress is computed from vertical scroll position relative to total readable content height.
- The header bottom border area is available and stable enough to host a 2px overlay indicator.

## Constraints

- Keep the indicator subtle but clearly visible (bright blue color, 2px height).
- Indicator must stay visible during scrolling.
- Implementation scope is only the book page reading view.

## Acceptance Criteria

- A 2px bright blue progress bar appears on the book page reading view.
- The bar is anchored over the header bottom border and remains visible while scrolling.
- The bar starts at the left edge and grows left-to-right as the user scrolls.
- At the end of reading scroll, the bar reaches full width and touches both borders.

## Dos

- Do implement smooth visual growth tied to scroll progress.
- Do ensure the bar positioning aligns with the header bottom border.
- Do keep the styling minimal and non-distracting.

## Don'ts

- Don't apply this indicator globally to unrelated pages.
- Don't change unrelated header behavior or layout.
- Don't introduce heavy animations or distracting effects.

## Open Questions

- Should progress be based on the full document height or a specific reading container if the book view uses one?
- Should the bright blue color use an existing design token/variable or a fixed hex value?
