# Improve LingoCafe Book Card Image-First Design

## Elevator's Pitch

Redesign the LingoCafe books catalog card so each card reads like a book cover instead of a content card. The image should become the whole card surface, with the book title and tags overlaid on top of it.

## Business Gain

The books catalog will feel more visual and more directly tied to the new plain background image assets. Users can scan available books faster because the card becomes a cover-first object instead of an image plus metadata block.

## Current State

The books page currently shows each book in a bordered card with a cover image at the top and a white content area below. The content area shows the title, author, SV/A2 indicator, and tags.

The operator has changed the book image to be a plain background image, which makes the current split card design feel wrong for the new asset style.

## Desired State

Each book card shows only the book image as the visible card surface. The card should have no visible border, while preserving rounded corners.

The title should be placed over the image, toward the top, as if it were printed on the book cover. A text style should be chosen and proposed that keeps the title readable over varying cover images.

Tags should also be shown over the image, close to the bottom.

The author should not be shown on the card.

The SV/A2 indicator should not be shown on the card.

## Definition of Success

The books catalog displays cover-like cards where the image fills the card, rounded corners remain, title text is readable near the top of the image, tags appear near the bottom of the image, and the old metadata block is gone.

## Additional Context

The operator supplied a screenshot of the current books catalog and clarified that the design should dramatically change because the image asset is now a plain background image.

The operator asked the implementation to pick and propose a title style that makes the title visible over the image.

## Assumptions

The target surface is the LingoCafe books catalog card shown on the `/books` page.

The card should remain clickable if it is currently clickable.

The title overlay can use a contrast treatment such as a subtle top gradient, shadow, high-contrast typography, or another implementation-specific style that keeps text legible without making the card look like the old bordered content layout.

The tags can use compact overlay pills or another readable overlay treatment, as long as they stay close to the bottom of the image.

## Constraints

Keep the card rounded corners.

Do not show a visible border.

Do not show the author.

Do not show the SV/A2 indicator.

Do not move the card design back toward a split image-plus-white-content card.

Keep the design responsive for the existing books grid.

Run `npm run qa` after implementation.

## Acceptance Criteria

- [ ] The books catalog card image fills the whole visible card surface.
- [ ] The card has rounded corners.
- [ ] The card does not show a visible border.
- [ ] The book title appears over the image near the top.
- [ ] The chosen title styling keeps the title readable over the provided book images.
- [ ] The author is not displayed on the card.
- [ ] The SV/A2 indicator is not displayed on the card.
- [ ] Tags appear over the image close to the bottom.
- [ ] The design remains usable on mobile and desktop grid layouts.
- [ ] Existing card click/navigation behavior is preserved.
- [ ] `npm run qa` passes after implementation.

## Dos

- Do make the card look like a finished book-cover surface.
- Do use an overlay treatment that protects title readability.
- Do place the title toward the top of the image.
- Do place tags close to the bottom of the image.
- Do preserve rounded corners.
- Do keep the card design visually clean.

## Don'ts

- Don't show the author.
- Don't show the SV/A2 indicator.
- Don't show a visible card border.
- Don't keep the old white metadata area below the image.
- Don't make the title unreadable on dark, bright, or busy image areas.
- Don't solve unrelated book detail or reader page design work in this task unless implementation discovers shared card code that must be adjusted.

## Open Questions

- Should the chosen title style prefer a book-cover typography look, a minimal app UI overlay, or a hybrid of both?
- Should tag overlay pills use a translucent background, solid background, or image-aware contrast treatment?
