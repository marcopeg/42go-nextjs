# Clarify Round v2 — Build pipx-installable 42Go Events CLI

## Questions

### 1. Distribution name

Is `42go-cli` acceptable as the Python package distribution name while the installed console command remains `42go`?

### 2. Existing script cleanup

Should the old event export/query scripts remain as compatibility shims, or should their logic move fully into the new `cli/` package?

### 3. Stats detail

Should `42go events query stats` include per-month counts?

### 4. Agent-navigable help

Should the CLI provide complete built-in help so an agent can navigate the command tree from `42go` and `--help` without separate context?

### 5. 42Go CLI skill

Should the repository maintain a dedicated `42go-cli` skill with a flat command index and richer sub-documents for complex commands?

## Answers

### 1.

Yes. Use `42go-cli` as the distribution name and keep `42go` as the console command.

### 2.

Move the logic fully into the `cli/` package.

### 3.

Yes. Include per-month counts.

### 4.

Yes. Invoking only `42go` should show help. Every command and subcommand should support standard help, most likely `--help`.

### 5.

Yes. Always keep the `42go-cli` skill up to date. The skill should contain a flat index of available commands with brief reasons to use each one. Complex commands should have dedicated sub-documents with full agent-facing documentation.

## Resolution Summary

- Package distribution name is `42go-cli`; console script remains `42go`.
- Event export/query logic should move fully into `cli/`; old standalone scripts should not remain as the primary implementation.
- `42go events query stats` should include per-month event counts in human-readable output.
- The CLI must be fully navigable by help: bare `42go` displays help and every command/subcommand supports standard `--help`.
- The task must create or update a `42go-cli` skill documenting the command tree for agents, with a flat command index and deeper docs for complex commands.
