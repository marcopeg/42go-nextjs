# Clarify Round v1 — Build pipx-installable 42Go Events CLI

## Questions

### 1. Package location

Should the Python CLI package live at the repository root or in a dedicated `cli/` folder?

### 2. Makefile relationship

Should `make events` stay as-is, or become a wrapper for `42go events pull`?

### 3. Stats output

For `42go events query stats`, should output be human table, JSON, or both?

## Answers

### 1.

Use a `cli/` folder.

### 2.

Make `make events` a wrapper around the CLI and clean up the current code.

### 3.

Use human-readable output.

## Resolution Summary

- The Python package should be implemented under `cli/`.
- `make events` should delegate to `42go events pull` after the CLI exists.
- Current event export/query script code should be cleaned up and integrated into the CLI instead of left as disconnected one-off scripts.
- `42go events query stats` should print human-readable output for the first version.
