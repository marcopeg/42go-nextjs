# Execution Notes — LingoCafe Data Model and Knex Migrations
**Task**: ./OG12.task.md
**Plan**: ./OG12.plan.md

## Decisions

- The raw `events` stream will keep `book_id` and `page_id` as nullable text references with no foreign keys so historical events survive disruptive re-exports.
- The partitioned `events` table will use a composite primary key on `(created_at, id)` because PostgreSQL requires the partition key to participate in parent-level primary-key enforcement.
- Seed data will intentionally be asymmetric: John is a fresh Lingocafe user with no profile or progress, while Jane is the returning demo user with a Swedish-learning profile and saved progress.
- The Lingocafe seed uses recognisable titles with fully original demo page text so the repository stays safe to distribute.

## Problems Encountered

- The sandbox could not reach the local PostgreSQL instance or Docker socket directly, so migration and seed validation had to be performed through escalated local commands.

## Deviations From Plan

- The task contract for `events.id` was tightened during execution so the partitioned table uses `PK(created_at, id)` instead of treating `id` alone as the primary key.
- The auth seed needed a small Lingocafe-specific cleanup path before deleting `auth.users` so repeated reseeds would not trip foreign-key constraints from `lingocafe.profiles` or `lingocafe.books_progress`.

## Additional Requests

- Added Lingocafe-specific seed coverage for John/Jane, four demo books, and asymmetric onboarding/resume state after the initial accepted plan was written.
- Added `target_level` and generic `data` storage to `lingocafe.profiles` so profile-driven filtering can grow without another schema fight.
- Reran the Lingocafe migration and seed flow after the profile change, then verified live in PostgreSQL that Jane's profile now stores `target_level = a2` and `data = {}`.

## Known Limitations

- No additional implementation limitations were discovered during this session. Event analytics beyond the raw stream still belong to later tasks.
