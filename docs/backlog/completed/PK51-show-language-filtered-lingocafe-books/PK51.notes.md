# Execution Notes — Show Language-Filtered LingoCafe Books
**Task**: ./PK51.task.md
**Plan**: ./PK51.plan.md

## Decisions

- Treat the operator's `backlog-execute ma68, pk51` request as plan acceptance plus execution.
- Use the existing `lingocafe.profiles` and `lingocafe.books` tables for the reader catalog.
- Return language options from `/api/lingocafe/books` and `/api/lingocafe/profile` so the client can render onboarding without component-local option arrays.
- Keep the language option lists as static API-local constants for now, per operator request.
- Use the current authenticated session user ID for profile reads/writes and never accept a browser-supplied user ID.
- Treat a profile as complete only when `own_lang`, `target_lang`, and `target_level` are all present.
- Keep the page in one of two states only: profile completion form or book list.

- Added `target_level` to onboarding persistence and made incomplete profiles return no book list until completed.
