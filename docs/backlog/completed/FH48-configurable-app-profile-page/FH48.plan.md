# Plan — Configurable App Profile Page

## Goal

Introduce an AppConfig-driven authenticated profile page that follows the public page block pattern, proves the block model with a minimal LingoCafe custom block, preserves the current profile diagnostics as configurable platform blocks, and starts ProfilePage documentation.

## Milestones

### Milestone 1 — Define Profile Block Contracts

- [x] Step 1 — Add profile block types to AppConfig
  - Achieve: Add `app.profile.items` to the app config model using typed block objects close to the public page pattern.
  - Create: `src/42go/components/ProfileBlock/types.ts` for `TProfileBlockItem`, platform block item types, custom component block types, save-result contracts, and block imperative handle/context types if needed.
  - Modify: `src/AppConfig.ts`.
  - Delete: n/a.
  - Touch points: `TAppConfigItem`, `app.profile`, public `TComponentBlock` pattern, client-only profile route constraints.
  - Validation: TypeScript accepts existing app configs without profile config; config authors get typed support for platform blocks and custom component blocks.
  - Notes: Keep the schema minimal enough to prove the pattern. Built-in platform blocks should use string `type` discriminants; custom app blocks should use direct React component references plus optional props.

- [x] Step 2 — Define save orchestration contracts
  - Achieve: Model the page-to-block save command so blocks can validate, persist, and report local/global feedback.
  - Create: Save result types such as validation success/error and persistence success/error in `src/42go/components/ProfileBlock/types.ts`.
  - Modify: n/a unless the renderer file owns these types instead.
  - Delete: n/a.
  - Touch points: profile page top-bar `Save preferences`, block-owned validation, block-owned persistence, page-level aggregation.
  - Validation: Types make it possible for a block to report a validation error before persistence and a persistence failure after validation.
  - Notes: The page must block persistence if any rendered block reports validation failure. Blocks without save behavior should return success/no-op.
