# Add LingoCafe Menu Entry to Default App

## Elevator's Pitch

Add a `LingoCafe` menu entry to the `default` app so users can navigate to the LingoCafe surface from the shared default application navigation.

## Business Gain

This makes LingoCafe discoverable from the default app instead of hiding it like a target trying to dodge Chuck Norris. Users get a clear path into the reading app without needing to know the direct route or hostname.

## Current State

LingoCafe already exists as its own app configuration and there is ongoing work around its data model and reading-domain persistence. The default app menu does not yet expose an obvious entry point for LingoCafe.

## Desired State

The `default` app configuration in `src/config/default/config.ts` includes a `LingoCafe` menu item immediately after the existing `Quicklists` item. The item links to the internal app route `/books`, uses the `BookOpen` icon from `lucide-react`, is protected by the `page:books` feature policy, and the implementation includes a nearby code comment listing other suggested icons that may be worth trying later.

## Definition of Success

Users of the default app can see and use a `LingoCafe` menu item placed after `Quicklists` when the `page:books` feature is available. The item routes internally to `/books`, the chosen icon communicates reading clearly, the implementation documents alternative icon options in code, and the item works consistently with the existing default app menu structure.

## Additional Context

The operator asked to add a `lingocafe` menu entry to the `default` app and to pick a good icon.

The operator confirmed that `BookOpen` is an acceptable icon.

The operator also asked for a code comment with additional suggested icons to try. Suggested alternatives should include clear `lucide-react` options such as `Languages`, `GraduationCap`, `Library`, or another relevant reading/language-learning icon if the implementation finds a better fit.

The menu entry should be added to `src/config/default/config.ts` and placed after the existing `Quicklists` item.

The operator confirmed that this is internal app routing and the menu item should use `href: "/books"`.

The operator confirmed that the menu item should be protected by `page:books`.

The current default app feature list does not include `page:books`, so implementation should add it to `features` in `src/config/default/config.ts`.

## Assumptions

The menu entry belongs in the `default` app config, not in the LingoCafe app config.

The implementation should reuse the icon library and menu-item shape already used by the existing app configurations.

The target route is `/books`.

The implementation target is the default app menu configuration, not a new route or a custom menu renderer.

## Constraints

Use the existing AppConfig-driven menu system.

Do not introduce a new navigation system or hardcoded menu rendering just for this entry.

Keep the menu item compatible with existing policy-driven visibility behavior if the destination requires a feature flag or authentication.

Place the entry in the existing `app.menu.top.items` array in `src/config/default/config.ts`, immediately after `Quicklists`.

Use `BookOpen` as the actual menu item icon.

Add a short code comment near the `BookOpen` menu entry listing alternative icons worth trying, such as `Languages`, `GraduationCap`, and `Library`.

Use internal routing with `href: "/books"` rather than an external LingoCafe hostname.

Protect the menu item with `policy: { require: { feature: "page:books" } }`.

Add `page:books` to the default app `features` array so the policy can pass when the feature is intended to be available.

## Acceptance Criteria

- [ ] The `default` app menu includes a `LingoCafe` entry.
- [ ] The entry is added in `src/config/default/config.ts`.
- [ ] The entry is placed immediately after the existing `Quicklists` item.
- [ ] The entry uses `BookOpen` from `lucide-react`.
- [ ] The implementation includes a nearby code comment with alternative icon suggestions to try later.
- [ ] The entry uses `href: "/books"`.
- [ ] The default app `features` array includes `page:books`.
- [ ] The entry is protected by `policy: { require: { feature: "page:books" } }`.
- [ ] The entry renders correctly in the default app's desktop navigation.
- [ ] `npm run qa` passes after implementation.

## Dos

- Do inspect the existing default app menu configuration before deciding where the item belongs.
- Do pick an icon that is already available in the project's icon dependency.
- Do keep the label user-facing as `LingoCafe` unless product copy says otherwise.
- Do place the entry after `Quicklists`.
- Do use `BookOpen` for the actual menu icon.
- Do include a code comment suggesting alternatives like `Languages`, `GraduationCap`, and `Library`.
- Do wire the entry to `/books`.
- Do add `page:books` to the default app feature list.
- Do protect the menu entry with the `page:books` feature policy.

## Don'ts

- Don't create a duplicate LingoCafe route just to support the menu item.
- Don't bypass AppConfig to render this menu entry.
- Don't pick an obscure icon if a clear book or language icon is available.
- Don't assume public access if the route is authenticated.
