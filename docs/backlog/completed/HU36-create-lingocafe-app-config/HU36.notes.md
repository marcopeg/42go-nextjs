# Execution Notes — Create LingoCafe App Config
**Task**: ./HU36.task.md
**Plan**: ./HU36.plan.md

## Decisions

- Added LingoCafe as a dedicated config module under `src/config/lingocafe/` to match the existing per-app structure.
- Used the shared `/login` page as the login entry point and configured the home page hero CTA to route there.
- Enabled `credentials` plus `google` providers and reused the existing shared Google environment variables.
- Set the post-login landing page to `/dashboard` so the shared login flow has a safe internal callback target.
- Added `lingocafe.localhost` alongside `lingocafe.app` for local development convenience while preserving the requested production hostname.
- Later aligned the app with the operator's updated config so the default landing page is `/books`, and added a basic authenticated books route with a fake reading list.

## Problems Encountered

- `npm run qa` initially failed inside the sandbox because Next.js attempted to fetch Google Fonts during the production build and could not reach `fonts.googleapis.com`.
- Direct smoke checks with `curl -H 'Host: lingocafe.app' http://localhost:3000/...` failed because no local server was listening on port `3000` during execution.

## Deviations From Plan

- Added `^lingocafe\\.localhost:\\d+$` alongside `^lingocafe\\.app$` to make local development possible without changing the requested production hostname.
- The final config was later adjusted by the operator to use the current LingoCafe host patterns and to land on `/books` instead of `/dashboard`.

## Additional Requests

- Added a very basic `/books` route after the config was changed to open there. The route uses `AppLayout` and renders a simple fake list of books to read.

## Known Limitations

- Google sign-in on `lingocafe.app` still depends on OAuth console settings allowing that host as an authorized origin and redirect domain.
- Host-routed runtime verification is still pending until the local app server is running.
- `npm run qa` passed again after the `/books` route was added, but no live browser smoke check was run in this completion step.
