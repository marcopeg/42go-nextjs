# Apply Profile Theme Selection Immediately

## Elevator's Pitch

Change the profile page theme selector so `light`, `dark`, and `system` apply immediately on click like a live preview and persist immediately in the existing theme system, instead of waiting for the profile page save action.

## Business Gain

Theme selection feels immediate and intuitive. The user sees the result at once, and the theme choice behaves like a device or UI preference instead of a delayed form field.

## Current State

The current core `ThemePreference` block on the profile page stages the selected theme as local draft state. The theme is only applied after the shared profile save action succeeds, which makes the selector behave differently from a normal preview-style theme switcher.

## Desired State

Clicking `light`, `dark`, or `system` on the profile page should immediately switch the app appearance and immediately persist that preference through the existing theme system. The user should not need to click the profile page save button for the theme choice to take effect, and the theme selector should no longer contribute to profile-page dirty state.

## Definition of Success

On the profile page, selecting a theme option immediately updates the active theme, refreshing or revisiting the app keeps the same choice through the existing persistence mechanism, and the theme selector behaves as an independent UI preference even though it still lives as a block on the profile page.

## Additional Context

Relevant core files already exist:

- `src/42go/components/ProfileBlock/blocks/ThemePreference.tsx` currently stages a draft theme and applies it only on `onSaveSuccess`.
- `src/42go/config/ThemeProvider.tsx` exposes the shared `useTheme()` integration backed by `next-themes`.
- `src/42go/components/ProfileBlock/ProfilePageRenderer.tsx` and `src/42go/profile/client.tsx` currently support save-driven dirty-state orchestration for the profile page.

This request is a follow-up to the earlier core theme selector work and changes the interaction model for that selector specifically.

## Assumptions

- The existing theme persistence path through the current theme system is still the correct storage mechanism.
- The theme selector should feel like a live UI preference control rather than a deferred form field.
- Other profile blocks may still rely on the shared profile save action.
- The theme selector remains visually placed as a block on the profile page even though its mechanic is independent from the real profile save flow.

## Constraints

- Keep using the existing shared theme system rather than introducing a second storage mechanism.
- Do not regress the rest of the profile page save behavior for non-theme blocks.
- Keep the change focused on the profile page theme selector behavior.

## Acceptance Criteria

- [ ] Clicking `light`, `dark`, or `system` on the profile page immediately applies that theme.
- [ ] The selected theme is immediately persisted through the existing theme persistence mechanism.
- [ ] Reloading or revisiting the app keeps the selected theme.
- [ ] The theme selector no longer depends on the profile page save button to apply the theme.
- [ ] Changing the theme does not mark the profile page as dirty and does not require any save action.
- [ ] Other profile page save behavior continues to work for the blocks that still need it.
- [ ] If immediate theme persistence somehow fails, the UI reverts to the previous theme.

## Dos

- Do preserve the shared `next-themes` integration as the source of persistence.
- Do keep the interaction feeling like a preview that becomes the actual selected value right away.
- Do remove or bypass dirty-state and save-state participation for the theme selector once the selection applies instantly.
- Do keep the revert path simple if local persistence unexpectedly fails.

## Don'ts

- Don't add a second persistence path for the theme.
- Don't make the user click `Save changes` just to see the theme apply.
- Don't break the save flow for other profile preferences.
- Don't let theme changes pollute the real profile save state.
