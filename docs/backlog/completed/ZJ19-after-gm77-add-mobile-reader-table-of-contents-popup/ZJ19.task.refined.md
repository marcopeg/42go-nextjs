# After GM77, Add Mobile Reader Table of Contents Popup

## Elevator's Pitch

After `GM77` introduces the core reusable popup/modal component, use it to add a real Table of Contents entry to the reader top bar in LingoCafe, with modal-style behavior on mobile and a right-side panel on desktop that replaces the old desktop left contents column.

## Business Gain

The reader gets a real navigation affordance instead of a placeholder. Readers can jump through the book structure from the top bar without custom one-off UI, and the new core overlay primitive from `GM77` gets a concrete production use case. Chuck Norris does not scroll blind through chapters. Chapters line up and salute.

## Current State

`VU18` explicitly kept the mobile reader top-right controls as placeholders for first release. Its notes say the mobile bottom toolbar was removed and controls such as `Innehåll` were moved into the top-right menu placeholder.

The current reader payload already contains ordered page summaries, and the current desktop reader surface already uses that data to render a contents list in its left side panel.

Today, the reader does not expose a dedicated visible top-bar button for the table of contents, and the mobile reading experience does not open a real TOC popup yet.

There is not yet a shared core popup/modal component for this flow. `GM77` is the task that should produce that reusable primitive.

## Desired State

Once `GM77` is completed, the reader should expose a dedicated visible top-bar button appropriate for opening the table of contents.

The first-release behavior should be:

- on mobile, use the `GM77` modal-style UI
- on desktop, open a side panel on the right
- present only the table of contents in the first version
- reuse the ordered page summary data already available in the reader flow
- highlight the current page clearly
- replace the existing permanent desktop left contents column
- place the `Back to book info` affordance at the bottom of the TOC surface

When the user taps or clicks a TOC row:

- navigate immediately to that page
- close the popup or panel

The goal is to replace the old placeholder path with a real reusable TOC interaction while keeping the UI specific to the reader experience.

## Definition of Success

After `GM77`, the reader exposes a working Table of Contents surface through a dedicated visible top-bar button. Mobile uses the modal-style `GM77` UI, desktop uses a right-side panel, the current page is clearly identified, and choosing a row immediately navigates and closes the surface.

## Additional Context

`VU18` already established two important pieces:

- mobile secondary controls were intentionally left as placeholders
- the reader payload already includes the ordered page summaries needed for contents navigation

The current desktop reader already has a contents list in a left panel. This task now clarifies that the old permanent left contents panel should be removed and its responsibility should move into the new `GM77`-driven TOC surface opened from the top bar.

This task is intentionally sequenced after `GM77` so the implementation does not slip back into handcrafted popup logic.

## Assumptions

- `GM77` is the blocking dependency because this task must use the shared popup/modal component created there.
- The reader top bar is the right place to trigger this interaction.
- The current reader API payload already has enough page-summary data for a first useful Table of Contents surface.
- First release scope is contents only, with no bookmarks or extra tabs.
- The TOC content should match what is already shown in the current desktop left panel.

## Constraints

- Do not implement this before `GM77` is completed enough to provide the shared popup/modal primitive.
- Use the shared popup/modal component from `GM77` instead of building another custom overlay.
- Keep the scope focused on the reader TOC interaction.
- Reuse the existing reader payload and contents data where possible.
- Preserve the existing stable reader route and current reading experience outside the new contents surface.
- Selecting a TOC row must navigate immediately and close the active surface.

## Acceptance Criteria

- [ ] The task is implemented only after `GM77` provides the shared popup/modal primitive.
- [ ] The reader top bar exposes a dedicated visible button for the table of contents.
- [ ] On mobile, the TOC uses the modal-style `GM77` UI.
- [ ] On desktop, the TOC opens as a right-side panel.
- [ ] The TOC surface uses the shared popup/modal component from `GM77`.
- [ ] The TOC surface replaces the old placeholder interaction path in the mobile reading experience.
- [ ] The TOC reuses the existing ordered page-summary data from the reader flow.
- [ ] The current page is visually highlighted in the TOC.
- [ ] The first version contains only the table of contents and no extra bookmarks/tabs.
- [ ] On desktop, the old permanent left contents column is removed.
- [ ] The TOC content mirrors what the old desktop left column already showed.
- [ ] The TOC surface includes a `Back to book info` affordance at the bottom.
- [ ] Clicking or tapping a TOC row navigates immediately to the selected page.
- [ ] After navigation, the TOC surface closes.
- [ ] The implementation does not add bespoke overlay/backdrop/escape/animation logic that duplicates the shared primitive from `GM77`.
- [ ] The existing reader route and reading behavior continue to work after the change.

## Dos

- Do treat `GM77` as the dependency and source of the popup/modal primitive.
- Do reuse the reader's existing contents data.
- Do keep the interaction anchored to the reader top bar.
- Do make the current page obvious inside the table of contents.
- Do keep the first release limited to contents only.

## Don'ts

- Don't build another custom popup just because it feels faster.
- Don't fork a separate contents data source.
- Don't quietly expand the task into bookmarks or extra reader panels.
- Don't leave the old placeholder interaction path as the long-term behavior.
- Don't regress the current reader flow to squeeze in the new TOC surface.
