# Clarify Round v2 — After GM77, Add Mobile Reader Table of Contents Popup

## Questions

### 1. Desktop already has a permanent left contents panel today. In the first version, should the new desktop right-side TOC panel replace that old left contents behavior, or should both coexist?

### 2. In the first version, should the TOC list show every page entry from the current `bookPage.pages` payload, or only a reduced chapter/section subset if available?

## Answers

### 1.

Replace the existing desktop left contents panel completely. Keep the full-page reading experience with the top bar, and move the contents/navigation responsibility into the new TOC surface.

### 2.

Show what is already shown in the existing desktop left bar. Use that as the TOC content model.

## Resolution Summary

- The new desktop right-side TOC panel should replace the old permanent left contents column.
- The main reader should stay as the full-page reading experience with the existing top bar.
- The TOC content should mirror what the old desktop left column already showed.
