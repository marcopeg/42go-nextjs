# Clarify Round v1 — After GM77, Add Mobile Reader Table of Contents Popup

## Questions

### 1. Where should the mobile TOC entry live in the first version?

Choices: keep it inside the existing `...` menu, or promote it to a dedicated visible top-bar button.

### 2. What should happen when the user taps a TOC row?

Choices: navigate immediately to that page and close the popup, or keep the popup open and require a second confirmation.

### 3. What is the intended first-release scope of the popup?

Choices: contents only, or contents plus placeholders such as bookmarks/tabs.

## Answers

### 1.

Use a dedicated visible top-bar button appropriate for the table of contents. Use the modal UI on mobile. On desktop, open a side panel on the right.

### 2.

Navigate immediately to the selected page and close the popup.

### 3.

The popup should contain only the table of contents. No bookmarks or other extra panels in the first version.

## Resolution Summary

- Promote the TOC affordance from the old `...` placeholder path to a dedicated visible top-bar button.
- Mobile should use the modal-style GM77 surface; desktop should use a right-side panel.
- Selecting a TOC item should navigate immediately and close the surface.
- First release scope is contents only.
