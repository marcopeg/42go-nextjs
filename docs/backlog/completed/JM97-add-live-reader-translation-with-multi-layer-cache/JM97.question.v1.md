### 4. How should sentence detection be implemented in the reader: extract from rendered DOM text nodes, wrap/tokenize sentences before rendering the markdown, or use a server-provided/pre-tokenized sentence model?

Probably the easy version is to wrap each sentence in a span or something like that with a non‑click. It might be the easier thing. Make sure that once tapped, it's also highlighted in the text. So I have a very good visual cue of what is being translated.

### 5. What normalization rules should feed the translation hash? For example: trim and collapse whitespace, Unicode NFC, preserve case, preserve punctuation, normalize language codes, or other rules.

Answer:

Apply all the normalizations that you can. The goal is not to translate the same meaning twice, probably lowercase, remove white spaces, something like that.
