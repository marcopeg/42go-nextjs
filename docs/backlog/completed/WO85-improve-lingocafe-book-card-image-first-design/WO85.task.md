# Improve LingoCafe Book Card Image-First Design

## Elevator's Pitch

Redesign the LingoCafe books catalog card so each card reads like a book cover instead of a content card. The image should become the whole card surface, with the book title and tags overlaid on top of it.

## Business Gain

The books catalog will feel more visual and more directly tied to the new plain background image assets. Users can scan available books faster because the card becomes a cover-first object instead of an image plus metadata block.

## Current State

The books page currently shows each book in a bordered card with a cover image at the top and a white content area below. The content area shows the title, author, SV/A2 indicator, and tags.

The operator has changed the book image to be a plain background image, which makes the current split card design feel wrong for the new asset style.

## Desired State

Each book card shows only the book image as the visible card surface. The card should have no visible border, while preserving rounded corners and the tall book-cover ratio.

The title should be placed over the image, toward the top, as if it were printed on the book cover. The chosen style should use book-cover typography: large serif or strong display type, white or cream text, a subtle shadow, and a dark top gradient to keep the title readable over varying cover images.

Tags should also be shown over the image, close to the bottom. Show at most three tags on the cover.

The author should not be shown on the card.

The SV/A2 indicator should not be shown on the card.

## Definition of Success

The books catalog displays cover-like cards where the image fills a tall cover-ratio card, rounded corners remain, title text is readable near the top of the image, up to three tags appear near the bottom of the image, mobile shows at least two covers per row, and the old metadata block is gone.

## Additional Context

The operator supplied a screenshot of the current books catalog and clarified that the design should dramatically change because the image asset is now a plain background image.

The operator asked the implementation to pick and propose a title style that makes the title visible over the image.

## Assumptions

The target surface is the LingoCafe books catalog card shown on the `/books` page.

The card should remain clickable if it is currently clickable.

The title overlay should use a dark top gradient, text shadow, and white or cream cover-style typography unless implementation discovers those exact choices fail against the provided assets.

The tags can use compact overlay pills or another readable overlay treatment, as long as they stay close to the bottom of the image and only the first three tags are shown.

## Constraints

Keep the card rounded corners.

Do not show a visible border.

Do not show the author.

Do not show the SV/A2 indicator.

Do not move the card design back toward a split image-plus-white-content card.

Keep the design responsive for the existing books grid.

Mobile must show at least two covers per row.

Preserve a tall book-cover ratio, roughly `2:3`, for cards.

Run `npm run qa` after implementation.
