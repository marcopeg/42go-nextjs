# Execution Notes — HeroBlock: Add Alignment Option
**Task**: ./NC71.task.md
**Plan**: ./NC71.plan.md

## Decisions

- Implemented alignment as explicit class/style maps in `HeroBlock`.
- Default `alignment` is `"center"` to preserve existing behavior.
- Left and right alignment move the max-width text blocks with `mr-auto` / `ml-auto`, align text, align actions, and adjust `backgroundPosition`.
- Documented `alignment` in the Hero Block content docs with a non-center example.
- Did not change live app sample configs; the docs example is enough for discoverability without forcing a visual change in an app.
- Used a temporary `src/config/home-page.tsx` visual fixture to verify left, center, and right alignment on `http://localhost:3000/`, then removed it.
- Reviewed `AK87` and `FE78` after implementation. `FE78` already mentions alignment behavior. `AK87` remains a thin draft and should still be refined on its own, but it does not need a direct update from `NC71`.
