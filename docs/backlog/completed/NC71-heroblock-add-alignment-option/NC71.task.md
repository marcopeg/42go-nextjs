# HeroBlock: Add Alignment Option


## Elevator's Pitch

Add the "alignment" (left, center, right) option to HeroBlock so it can be applied to the contents. Optional, default "center".

## Business Gain

Public page authors can tune HeroBlock composition from configuration without creating app-specific hero components. This makes landing pages and documentation pages more flexible while keeping the shared ContentBlock system simple.

## Current State

`HeroBlock` renders its content centered. The section, text, max-width content, and action buttons are all visually centered. `backgroundImage` support exists, but its background position is fixed to center.

## Desired State

`HeroBlock` accepts an optional `alignment` property with allowed values `"left"`, `"center"`, and `"right"`.

When omitted, `alignment` defaults to `"center"` and existing HeroBlock configurations render as they do today.

When set to `"left"` or `"right"`, alignment applies to the full hero composition:

- title and subtitle text alignment
- action button row justification
- content column origin inside the hero container
- background image starting origin, using an alignment-appropriate background position

## Definition of Success

Page configuration can align HeroBlock content left, center, or right without custom components. Existing heroes remain backward compatible. The option is typed, implemented, and documented in the Hero Block content documentation.

## Additional Context

The current component is `src/42go/components/ContentBlock/blocks/HeroBlock.tsx`.

Hero Block documentation currently exists under `contents/default/docs/composable-pages/hero-block/README.md`.

## Assumptions

- `"center"` is the default because it matches the current rendering.
- Alignment is a HeroBlock configuration concern, not a new layout system.
- `backgroundImage` alignment means changing the image origin/position consistently with content alignment.

## Constraints

- Keep the task focused on HeroBlock alignment only.
- Preserve existing HeroBlock behavior when `alignment` is missing.
- Do not bundle optional field work from `AK87` or icon support from `FE78`.
- Update documentation in `contents/default/docs/composable-pages/hero-block/README.md`, or add equivalent Hero Block documentation under `contents` if the location changes before implementation.
- Follow existing ContentBlock type and rendering patterns.

## Acceptance Criteria

- `THeroBlock` supports `alignment?: "left" | "center" | "right"`.
- Missing `alignment` behaves as `"center"` and keeps current visual output.
- `alignment: "left"` aligns title, subtitle, action row, content origin, and background image origin to the left.
- `alignment: "center"` aligns title, subtitle, action row, content origin, and background image origin to the center.
- `alignment: "right"` aligns title, subtitle, action row, content origin, and background image origin to the right.
- Existing HeroBlock configurations compile and render without changes.
- Hero Block documentation describes the `alignment` option and includes at least one non-center example.
- After implementation, review `AK87` and `FE78` to decide whether their drafts need updates based on the alignment outcome.

## Dos

- Keep the API configuration-driven.
- Implement alignment with explicit, readable class/style mapping.
- Keep background-image mode compatible with existing overlay behavior.
- Prefer a narrow change in the shared HeroBlock component and its docs.

## Don'ts

- Do not redesign HeroBlock beyond alignment.
- Do not make `alignment` required.
- Do not change unrelated ContentBlock APIs.
- Do not implement `AK87` or `FE78` as part of this task.
