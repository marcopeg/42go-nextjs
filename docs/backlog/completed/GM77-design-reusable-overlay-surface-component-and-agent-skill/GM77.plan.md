# Plan — Design Reusable Overlay Surface Component and Agent Skill

## Goal

Build a shared 42Go `Modal` component that uses Radix Dialog behavior where practical, supports mobile full-screen presentation and desktop `modal` / `panel` presentations through a simple prop API, documents the component through a `42go-ui-modal` agent skill, exposes demo/styleguide routes for `Panel` and `Modal`, and proves reusability by replacing the LingoCafe reader appearance preferences popup mechanics on both mobile and desktop where practical.

## Milestones

### Milestone 1 — Establish The Modal Foundation

- [x] Step 1 — Add or confirm the Radix Dialog foundation
  - Achieve: Provide the accessible dialog lifecycle foundation for `Modal`, including semantic dialog structure, Escape handling, focus containment/restoration, portal rendering, and outside-interaction behavior.
  - Create: A shadcn-style dialog wrapper if the implementation chooses to keep low-level Radix primitives in `src/components/ui/dialog.tsx`.
  - Modify: `package.json`, `package-lock.json` to add `@radix-ui/react-dialog` if no existing dependency covers the need.
  - Delete: Nothing.
  - Touch points: `package.json`, `package-lock.json`, `src/components/ui/`, Radix Dialog primitives, existing `src/components/ui/dropdown-menu.tsx` conventions for wrapper style.
  - Validation: `npm install` or equivalent dependency update succeeds; TypeScript can import the chosen Dialog primitives; no invalid dependency tree remains.
  - Notes: Use Radix as much as possible. Do not build a custom focus trap unless Radix cannot cover a specific requirement.

- [x] Step 2 — Decide Modal vs Panel code boundaries
  - Achieve: Make the implementation decision on whether `Modal` reuses existing static `Panel` pieces internally or only matches their visual language.
  - Create: No required files unless a tiny shared visual helper is justified.
  - Modify: Potentially `src/42go/components/panel/` only if a small non-breaking export is needed; otherwise keep it untouched.
  - Delete: Nothing.
  - Touch points: `src/42go/components/panel/Panel.tsx`, `src/42go/components/panel/SimplePanel.tsx`, `src/42go/components/panel/PanelHeader.tsx`, new `src/42go/components/modal/`.
  - Validation: The static `Panel` still behaves as before; `Modal` owns all overlay lifecycle and does not blur responsibility with `Panel`.
  - Notes: `Panel` is a static content shell. `Modal` is the overlay/lifecycle component. Chuck Norris knows the difference.
