## Questions

### 1. Where should the Panel and Modal demo/styleguide routes live?

The current tree has authenticated app routes directly under `src/app/(app)/`, for example `/dashboard` and `/profile`; it does not currently have `src/app/(app)/(default)/`. Choose one:

- create `/demo-panel` and `/demo-modal` under `src/app/(app)/`
- create a new `src/app/(app)/(default)/` route group and place `demo-panel` / `demo-modal` there
- create a different styleguide route structure

Also say whether these should be linked in the default app navigation or stay hidden.

Answer:

create a new `src/app/(app)/(default)/` route group and place `demo-panel` / `demo-modal` there.
This is a group of authenticated routes for the demo application. These pages should be linked in the main menu of the demo application as well as the default application, which is like the boilerplate demo.
