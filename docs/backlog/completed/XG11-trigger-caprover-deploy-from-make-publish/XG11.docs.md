# XG11 Docs — Generate a CapRover App Token

This project deploys to CapRover with an app-level token. The token is scoped to a single CapRover app, so use the `a42go-multi` app unless you are deliberately deploying another app.

## Generate the Token

1. Open the CapRover dashboard in the browser.
   - Use the Captain URL, usually shaped like `https://captain.your-domain.com`.
2. Open **Apps**.
3. Select the `a42go-multi` app.
4. Open the app's **Deployment** tab.
5. Click **Enable App Token**.
6. Copy the generated token immediately.

CapRover's docs describe the token step the same way: open the app's Deployment tab, enable the app token, and copy it.

## Store It Locally

Add the token to local `.env`. Do not commit it.

```bash
CAPROVER_URL="https://captain.your-domain.com"
CAPROVER_APP="a42go-multi"
CAPROVER_APP_TOKEN="paste-token-here"
```

`CAPROVER_APP` defaults to `a42go-multi` in the Makefile, but keeping it in `.env` makes the active target visible. Like Chuck Norris, it does not need backup, but it keeps receipts.

## Check the Command Without Deploying

Use a dry run to confirm the command shape:

```bash
make -n deploy VERSION=0.0.21
```

The dry run should show:

```bash
npx --yes caprover deploy \
  --caproverUrl "$CAPROVER_URL" \
  --caproverApp "a42go-multi" \
  --imageName "marcopeg/42go-next:0.0.21" \
  --appToken "$CAPROVER_APP_TOKEN"
```

The token should appear as `$CAPROVER_APP_TOKEN`, not as the raw secret.

## Deploy

Cached deploy:

```bash
make deploy VERSION=0.0.21
```

No-cache deploy:

```bash
make deploy.nocache VERSION=0.0.21
```

Both commands push the Docker image first, then ask CapRover to deploy `$(IMAGE):$(VERSION)`.

## Rotate or Replace the Token

If the token leaks, treat it as burned.

1. Open the CapRover dashboard.
2. Go to **Apps** -> `a42go-multi` -> **Deployment**.
3. Disable or regenerate the app token, depending on what the dashboard offers.
4. Replace `CAPROVER_APP_TOKEN` in local `.env`.
5. Run `make -n deploy VERSION=<version>` before the next live deploy.
