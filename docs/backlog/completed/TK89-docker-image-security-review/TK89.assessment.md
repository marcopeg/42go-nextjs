## Remaining Findings

The remaining work is not the same kind of problem as the original release blockers. The image itself is clean from Trivy's runtime vulnerability perspective. The remaining items are policy and operational decisions:

1. Local dump files remain in the repository tree and contain token-looking material. They are excluded from Docker build context and final image, but they are still a local repository hygiene risk.
2. `docker-compose.prod.yml` still reports local-production warnings. These are not real production blockers because the file is explicitly for local production mimicry only.
3. Image-level health check is disabled. This is acceptable if the deployment platform owns health checks, but the choice should be documented.
4. Docker Hub publishing still lacks SBOM, provenance, signing, and a written `latest` tag movement policy.

## Assessment

The remediation already performed fixed the actual runtime blockers: vulnerable Next.js package state, redundant dependency layer, package-manager tooling in the runtime image, source-map exposure, and build-context leakage. The current image has passed local full security assessment and operator production smoke testing.

The next decisions are about distribution posture and operations. For going live on the current VPS deployment, there is no Docker image security blocker left. If the image is meant to be broadly consumed from Docker Hub, supply-chain metadata becomes a policy decision rather than a vulnerability blocker.

The local dump/token issue is separate from Docker publishing. Docker no longer ships it. The repository still carries it. The current direction is to defer this to a backup-system review and likely move dump outputs into `.local/`, following the existing events dump pattern. Token rotation is only necessary if those dump files left this machine or entered shared infrastructure.

Source-map follow-up: the Dockerfile now deletes runtime source maps from `.next/standalone` and `.next/static` immediately after `next build`, before the final runner image copies those files. Rebuilt image verification returned `0` source maps under `/app`.
