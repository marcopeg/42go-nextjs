# Execution Notes — Split Bookshelf Into Reading and Alphabetical Catalog Sections
**Task**: ./QL90.task.md
**Plan**: ./QL90.plan.md

## Decisions

- Use `readingAction.kind === "resume"` plus a string `href` as the currently-reading signal.
- Use `lingocafe.books_progress.updated_at` as the source for most-recently-read ordering.
- Keep `BookCard` responsible for link behavior so grouping does not change navigation.
- Keep catalog ordering client-side so the API can retain its existing language-filtered payload and level-first default order for any other consumers.
- Do not render an empty `Catalog` section if every visible book is already in the `Currently Reading` section.

## Problems Encountered

- The first `npm run qa` failed because the shared `ReaderBookReadingAction` type gained `updatedAt`, and the book-detail page had its own unavailable-action fallback. The fallback was updated to include `updatedAt: null`.

## Deviations From Plan

- Operator accepted the plan and added a new execution-time requirement: sort `Currently Reading` by most recently read first instead of alphabetical title order.
