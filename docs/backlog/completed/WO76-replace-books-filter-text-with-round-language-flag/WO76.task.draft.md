# Replace Books Filter Text With Round Language Flag

## Elevator's Pitch

On the LingoCafe `/books` page, replace the current subtitle text such as `Catalog filtered for Swedish.` with a round language flag shown on the right side of the books header bar.

## Business Gain

The books header becomes more visual, compact, and language-focused. Users can understand the active reading language at a glance without burning vertical space on a sentence. Chuck Norris does not read a status paragraph when a flag can finish the job.

## Current State

The current books page renders `Books` as the title and uses the subtitle line to show text like `Catalog filtered for Swedish.` when `targetLang` is available. This comes from `src/app/(app)/(lingocafe)/books/page.tsx`, where `targetLabel` is converted into the subtitle string. The provided screenshot shows that text under the `Books` title.

## Desired State

Instead of showing the `Catalog filtered for ...` sentence, the books header should show the active learning language as a round flag icon aligned on the right side of the books header bar. The title `Books` should remain, but the textual filter sentence should be removed for this state. The visual treatment should match the screenshot intent: cleaner header, compact flag, and no extra explanatory text.

## Definition of Success

A user landing on `/books` can immediately identify the active reading language from a round flag badge in the header area, and the old `Catalog filtered for ...` subtitle text is no longer shown.

## Additional Context

The operator provided screenshot `./.hal/users/7974709349/uploads/image_1778258802193.jpg`, showing the current header with `Catalog filtered for Swedish.` under `Books`. The requested change is specifically to replace that text with a language flag on the right side of the bar. This is narrower than `BL01`, which is a broader draft about a full top-bar language-and-level switcher via `GM77` popup behavior.

## Assumptions

- The active flag should represent the current `targetLang` / learning language, not `ownLang`.
- This request is primarily visual and does not yet require a popup, panel, or language-switching interaction.
- The existing books data/profile load already provides enough information to determine the active language for rendering the flag.
- If the language is unavailable, implementation can fall back gracefully rather than rendering broken UI.

## Constraints

- Keep the task scoped to the books header presentation.
- Do not broaden this task into the full `BL01` language-and-level switcher flow.
- Reuse the current profile/books page state instead of creating a second fetch path.
- Do not regress onboarding, books loading, or catalog rendering.
- Keep the implementation aligned with the authenticated app-page conventions already used by `/books`.

## Acceptance Criteria

- [ ] On `/books`, the header no longer shows subtitle text in the form `Catalog filtered for ...` when the language flag is available.
- [ ] The `Books` header area shows a round language flag aligned to the right side of the bar.
- [ ] The flag represents the active learning language derived from the existing books/profile state.
- [ ] The header remains visually clean and compact on mobile, matching the screenshot intent.
- [ ] The change does not introduce a popup or full language-switching flow as part of this task.
- [ ] Onboarding, loading, and catalog rendering behavior are not regressed.

## Dos

- Do treat this as a focused header-UI change.
- Do reuse the existing `targetLang`/profile information already available on the books page.
- Do keep the flag visually compact and clearly legible.
- Do keep the mobile header layout stable.

## Don'ts

- Don't keep the old `Catalog filtered for ...` sentence alongside the flag in the final header state.
- Don't turn this task into the full `GM77` popup switcher flow.
- Don't add a second API or profile-loading path for the same information.
- Don't regress the rest of the books page just to restyle the header.

## Open Questions

- What should the header show when no active learning language is available: no flag, a neutral placeholder, or the existing fallback copy?
- Should the round flag be purely visual in this task, or should it already behave like a button for future switcher work?
