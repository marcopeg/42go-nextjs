# Execution Notes — Display Book Covers
**Task**: ./DG66.task.md
**Plan**: ./DG66.plan.md

## Decisions

- Treat the accepted plan as execution approval and move `DG66` into WIP.
- Keep the two operator-provided covers exactly as supplied. They are 500x750 JPEG files under `public/images/lingocafe/` and their filenames match seeded book IDs.
- Use `/images/lingocafe/<book-id>.jpg` as the default public cover URL and `/images/lingocafe/placeholder.jpg` as the generic fallback.
- Ignore remote `books.cover` URLs for now so `next/image` does not require remote image domain configuration.
- Leave seed `cover: null` values intact so default book-id cover inference is exercised.
- Render book covers with `next/image` in a fixed 2:3 frame and handle failed image loads by swapping to the generic placeholder.
- Keep the books-list API lean by excluding `description` from the selected book fields and from the rendered card UI.
- `npm run qa` passed after removing descriptions from the API and UI.
- The available cover files are 500x750, so the UI uses a `2:3` cover frame. The mobile card now gives covers a larger responsive width and uses `object-contain` to preserve the full artwork without cropping.
- Fixed the mobile layout after the larger cover frame introduced horizontal overflow. Mobile cards now stack with the cover on top of the book information, while larger viewports keep the side-by-side layout.
- Replaced the viewport-based mobile cover width with `w-full max-w-full`, so the fixed `2:3` cover frame is sized by the available card width. The mobile title/badge row also stacks to avoid forcing card overflow.
- The remaining horizontal scroll came from the global mobile drawer, not the book image. The closed drawer was always mounted as a fixed `w-4/5` off-canvas panel translated to the right, which extended the page's scrollable width. The drawer is now mounted only while open.
- Authenticated in-app browser smoke on `http://localhost:3000/books` as `admin/admin` confirmed the catalog renders correctly and the horizontal scroll is gone after the drawer mount fix.
- `npm run qa` passed after the final `AppLayout` drawer change.
