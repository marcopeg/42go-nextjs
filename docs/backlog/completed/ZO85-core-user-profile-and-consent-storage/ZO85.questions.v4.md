## 3. AJV Details

JSON Schema "draft" means the version of the JSON Schema spec AJV validates against. AJV v8 commonly supports draft-07 by default and also supports newer drafts with different imports/config. Draft-07 is boring and well-known. Newer drafts have more expressive keywords but more moving parts.

Strict mode catches schema mistakes early, but may reject loose schemas and make config authoring more annoying.

Questions:

## 5. Consent Legal Metadata

`purpose`, `legalBasis`, and `category` are optional metadata that can describe why the consent/acknowledgement exists. Examples:

- `purpose: "Accept Terms and Conditions"`
- `legalBasis: "contract"` for terms required to provide the service
- `legalBasis: "consent"` for marketing opt-in
- `category: "legal" | "privacy" | "marketing" | "program"`

This is not a legal opinion. It is a way to keep app config explicit instead of burying meaning in `name`.
