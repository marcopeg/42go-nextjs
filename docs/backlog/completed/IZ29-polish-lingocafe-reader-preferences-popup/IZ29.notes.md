## Decisions

- The operator invoked `backlog-execute IZ29`, treated as accept + execute.
- Added the operator's timestamp requirement to the task and plan before reader UI implementation: backlog timestamps touched from this point forward should use UTC ISO-8601 with `Z`.

- Replaced the reset confirmation `Modal` with browser-native `window.confirm()` after operator feedback. This keeps the confirmation small and avoids another modal scroll-lock layer.

## Problems Encountered

- A direct `py_compile` attempt tried to write to the existing `.agents` `__pycache__` path and hit a sandbox permission error. Retried with `PYTHONPYCACHEPREFIX=/private/tmp/pycache-42go`, which succeeded.
- The initial confirmation used a nested `Modal`. Operator testing showed that after reset and close, scrolling stopped in both the reading settings modal and the chapter page. The likely cause was nested modal scroll locking remaining active.
- Browser verification through the Codex in-app browser reached `https://lc42go.ngrok.app/login`, but submitting `john`/`john` and `john.doe@example.com`/`john` stayed on `/login` without visible page progress. The browser was made visible to the operator for inspection.
- The operator later clarified the ngrok login failure happened while a production database restore was active, and that database did not contain the `john` test user. The operator also reported that `john`/`john` works in Chrome Canary after restoring the demo/development database, while the embedded Codex browser still appeared unreliable for that auth flow.

## Deviations From Plan

- Added a new UTC timestamp milestone before the original reader preferences UI milestones because the operator requested it before execution.
- Changed confirmation from an app-native nested modal to browser-native `confirm()` because the operator explicitly allowed it and it removes the scroll-lock regression path.

- Operator requested the reset confirmation be a small inline popup, with JavaScript `confirm()` acceptable for now.
- Operator requested the device-storage note be reduced from a bordered card with joke copy to a very subtle bottom line only.
