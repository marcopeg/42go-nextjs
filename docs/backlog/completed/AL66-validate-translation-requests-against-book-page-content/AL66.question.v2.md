## Questions

### 1. Single-sentence full-stop rule: accept the strict v1 guard?

My recommendation: for this backend guard, keep v1 intentionally strict and cheap.

- Trim and normalize whitespace first.
- Reject any `.` that is not the final character.
- Allow zero full stops, because the reader may send sentences ending in `!`, `?`, quotes, or no punctuation after extraction.
- Allow one terminal `.`, possibly followed only by closing quotes/brackets if implementation can do that simply.
- Do not add abbreviation, decimal, or ellipsis exceptions in this task.

This can reject legitimate sentences like `Dr. Jekyll walked home.` but avoids a fragile exception dictionary and matches the goal: stop modified clients from sending paragraphs or arbitrary text to the provider. The page-membership check still catches real book text after cache misses, but this cheap gate remains deliberately conservative.
