# Refinement Questions v1 - Validate Translation Requests Against Book Page Content

## Questions

### 1. Payload shape: should the backend keep using `pageId`, or introduce `chapterId`?

The reader currently sends `bookId` and `pageId`, and `loadBookPage(bookId, pageId)` already maps to `lingocafe.books_pages`. Should implementation keep `pageId` as the required chapter/page identifier, add a new `chapterId`, or accept both?

Answer:

I think we are generally using the concept of "page". 
If you don't have any hard motivation to push for a change, let's stick to it.

### 2. Cache-hit validation: should cached translations skip page-membership checks?

The draft currently says cheap text validation runs first, then a memory/database cache hit may return without loading the chapter/page. Only uncached text must prove it exists in the requested chapter before provider use. Is that the intended rule?

Answer:

I'd say:

1. formal validation to drop obvious abuses with low impact (no db check)
2. chache hit check 
3. validate request against the source page
4. upstream provider hit for real translation

### 3. Single-sentence rule: how strict should full-stop validation be?

Should the first implementation reject any internal `.` and allow only zero or one terminal `.`, even if that rejects abbreviations, decimals, and ellipses? Or should it support any exceptions now?

Answer:

Good question. stuff like "dr. jackyll" would return a false positive.
But I think building an extensive dictionary of exception can be very complex.

What do you suggest?

### 4. Content matching: which chapter/page surfaces count, and how forgiving should normalization be?

The current task includes title, summary, and markdown body content. Should the backend match against all three after whitespace/markdown normalization, or only the body content? Should matching require exact normalized sentence text, or allow looser matching?

Answer:

It should include title and summary, and a looser check is ok. 
The goal is to verify that we are really translating stuff from the book and not abusing the system.
