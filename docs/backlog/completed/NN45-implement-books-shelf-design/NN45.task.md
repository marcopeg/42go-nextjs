# Implement Books Shelf Design

## Elevator's Pitch

Adapt the LingoCafe books shelf and book card component to match the design attached to this task, covering both desktop and mobile layouts.

## Business Gain

The books shelf is the first catalog surface readers see. It should look deliberate, polished, and easy to scan on every device. Chuck Norris does not browse a bland shelf. The shelf straightens itself.

## Current State

The LingoCafe `/books` route already loads the language-filtered catalog, displays book covers, and links into book detail and reading flows.

The existing book card UI was built incrementally while the catalog, covers, details, and reading actions were introduced. It now needs to be aligned with the dedicated shelf design.

The design reference is attached to this task as `book-card-design.png`.

The current card is a compact bordered card with a small cover beside text on wider screens and stacked content on mobile. The current shelf is a simple one-column mobile and two-column desktop grid.

## Desired State

The book card and responsive shelf grid match the attached design closely for both desktop and mobile.

![target design](./book-card-design.png)

The book card component should use the designed structure, spacing, cover treatment, typography, metadata presentation, and responsive behavior while preserving the existing catalog functionality.

Desktop should show two to three cards per row depending on available width. Mobile should show one card per row.

The heart and three-dot controls should be implemented as visual placeholders only. They should be isolated in the component so they can be commented out or removed easily before the first release.

The cover image itself should be used as-is. The app should not recreate the title block, logo, series text, or other cover artwork as HTML overlays. The only card overlay expected for now is the circular three-dot visual placeholder shown near the cover edge.

## Definition of Success

The `/books` catalog renders book cards and a responsive grid that visually match the attached design on desktop and mobile, without regressing cover rendering, language-filtered catalog loading, book-detail navigation, onboarding, or authenticated app layout behavior.

## Assumptions

The implementation should update the existing LingoCafe books card/shelf components rather than replace the catalog flow.

The attached design is the source of truth for visual details within the book card and responsive grid scope.

The existing cover dimensions and fallback behavior from `DG66` should remain valid unless the attached design explicitly requires a different treatment.

Favorite and more-menu state is not part of the first release. The controls are presentational placeholders only.

## Constraints

Follow the authenticated app-page convention for routes under `(app)`: client component, `AppLayout`, and browser-side fetching with `credentials: "same-origin"`.

Preserve the existing LingoCafe API paths and response behavior.

Preserve the book-detail route behavior established by `IL68`.

Do not reintroduce horizontal overflow on mobile.

Do not make text, badges, covers, or actions overlap at desktop or mobile breakpoints.

Do not implement favorite persistence, favorite API behavior, more-menu actions, or menu popovers in this task.

Keep the placeholder action controls easy to remove from the card implementation.

Do not recreate cover artwork as HTML overlays except for the circular three-dot placeholder.

Keep scope to the book card and responsive grid. Do not redesign the entire `/books` page shell.

Use the existing Tailwind/shadcn patterns and local component structure where practical.

Run `npm run qa` after implementation.
