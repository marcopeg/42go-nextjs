# Execution Notes — Implement Book Details Page Design
**Task**: ./DJ89.task.md
**Plan**: ./DJ89.plan.md

## Decisions

- Treat the operator's "yes an execute" response as plan acceptance plus execution approval.
- Move `DJ89` into WIP before code changes.
- Reuse `NN45` shelf implementation decisions: keep cover artwork as image content, isolate placeholder actions, use route-local LingoCafe book components, and avoid oversized desktop layouts.
- Keep the existing mobile full-screen detail shell and focus on the content inside it.
- Add Contents data through the existing `/api/lingocafe/books/[bookId]` detail payload rather than creating a separate endpoint.
- Do not fetch full page content for the detail Contents section; use ordered page summary rows and read-page hrefs.
- Keep secondary detail actions as isolated visual-only placeholders with no persistence, popovers, menus, or API calls.
- Use basic English labels such as "Contents", "Show all chapters", and "About this book"; app-level i18n remains out of scope.
- Removed the desktop breadcrumb from the detail content after operator visual review.
- Removed the centered `mx-auto` wrapper from `BookInfoContent` so desktop content starts at the same content edge as the top-bar title.
- Added a desktop-only left offset to `BookInfoContent` so the content aligns with the toolbar title/subtitle column instead of the back-button column.
- Removed the duplicate language/level pill above the detail title.
- Reduced the desktop cover columns from 18/20rem to 14/16rem.
- Removed the mobile carousel dots because a book has only one cover.
- Fixed mobile horizontal overflow by clamping the mobile shell, scroll area, content wrapper, grid, and cover to the viewport width.
- Changed the app toolbar title to `All books` to avoid duplicating the selected book title above the main detail heading.
- Moved level, language, and page count into the cover/left column below the cover.
- Moved `About this book` into the main/right column above the CTA row.
- Moved `Contents` into the main/right column below the CTA row.
- Made the cover/details left column sticky on desktop with `md:sticky md:top-24`; mobile remains normal flow.
- Updated mobile only: cover now uses the available content width with square edges and no border, while desktop keeps rounded bordered cover styling.
- Added a mobile-only three-column metadata row before tags with icon/value pairs; desktop metadata remains in the sticky left column.
- Aligned the mobile metadata row columns so level is left aligned, language is centered, and page count is right aligned.
