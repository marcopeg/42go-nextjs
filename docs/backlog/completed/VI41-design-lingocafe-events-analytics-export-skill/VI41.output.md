# Output — Design LingoCafe Events Analytics Export Skill

## Decision

Use a download-only local analytics archive for `lingocafe.events`.

The VPS PostgreSQL database is treated as a live event buffer. The durable analytics archive lives on the local filesystem and stores each export batch twice:

- CSV for human-readable inspection.
- Parquet for Python and DuckDB analysis.

The skill does not delete, truncate, retain, or clean up source rows.

```bash
export LC_EVENTS_DATABASE_URL="postgres://user:pass@host:5432/db"
```

Optional archive root override:

```bash
export LC_EVENTS_ANALYTICS_DIR=".local/lingocafe-analytics"
```

Default archive root:

```text
.local/lingocafe-analytics
```

## Local Layout

```text
.local/lingocafe-analytics/
  events/
    csv/batch-YYYYMMDDTHHMMSSZ.csv
    parquet/batch-YYYYMMDDTHHMMSSZ.parquet
    state.json
    manifest.jsonl
    inflight.json
```

`inflight.json` is temporary. It lets an interrupted run reuse and overwrite the same incomplete batch before the cursor moves.

## Cursor Rules

The export cursor is `(created_at, id)`.

First run:

```sql
SELECT ...
FROM lingocafe.events
ORDER BY created_at ASC, id ASC
LIMIT :limit
```

Incremental runs:

```sql
SELECT ...
FROM lingocafe.events
WHERE (created_at, id) > (:last_created_at, :last_id)
ORDER BY created_at ASC, id ASC
LIMIT :limit
```

`event_at` is used for behavior analytics. It is not used for export progress.

## Commit Rules

`state.json` advances only after:

1. CSV output is written.
2. Parquet output is written.
3. DuckDB can read the Parquet output and count the expected rows.
4. `manifest.jsonl` records the completed batch.

If a run dies before that point, rerunning reuses the incomplete batch ID and overwrites the incomplete output.

## Analysis Command

Weekly active users:

```bash
python3 .agents/skills/lingocafe-events-export/scripts/analyze_events.py wau
```

The command reads local Parquet batches only. It does not query PostgreSQL.

## Reading Session Process

The README documents the first session-analysis assumption:

- Use `event_at` for behavioral ordering.
- Group by `user_id` and `book_id`.
- Start a new session when the gap exceeds 30 minutes.
- Use bounded event gaps for early time-spent estimates.
