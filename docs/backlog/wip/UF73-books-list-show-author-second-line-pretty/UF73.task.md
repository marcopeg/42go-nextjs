# Books List: Show Author as Stylish Second Line

## Elevator's Pitch

In the books list UI, show each book author as a second line directly under the title using the format `by {author}`. Keep it visually polished and readable.

## Business Gain

Improves scannability and perceived quality of the books list, helping users identify books faster and making the interface feel more premium.

## Current State

On `/books` (Lingocafe base URL), list rows do not yet present the author as a deliberate second-line subtitle in the requested format.

## Desired State

On `/books` (Lingocafe base URL), every row displays:
- First line: book title
- Second line: `by {author}` (only when author exists)

If author is missing, the second line is omitted entirely.

Visual direction is **A: subtle/clean** — smaller muted secondary text with balanced spacing and strong readability in all themes and viewport sizes.

## Definition of Success

Users can instantly scan title + author where available, while rows with missing author remain visually clean without placeholder noise. The subtitle treatment looks intentional, subtle, and consistent.

## Additional Context

Operator clarified:
- Target route: `/books` (Lingocafe base URL)
- Missing author behavior: do not show a second line
- Style choice: A (subtle/clean muted subtitle)

## Assumptions

- `/books` list items provide optional author data or an equivalent field that can be conditionally rendered.
- Existing design system tokens/components should be reused where possible.

## Constraints

- Keep existing `/books` list behavior intact (selection, navigation, actions).
- Do not render author subtitle when author is missing.
- Styling must follow app theme tokens and the subtle/clean direction (no flashy accents).
- Avoid introducing unnecessary complexity.

## Acceptance Criteria

- On `/books`, each item with an author shows a second line under title in the format `by {author}`.
- Items without an author render no second line (no fallback text).
- The author line uses a subtle/clean hierarchy (smaller, muted text; consistent spacing).
- The layout remains responsive and preserves row alignment with mixed author/no-author rows.
- The styling is readable in supported themes without contrast issues.

## Dos

- Reuse existing typography and spacing conventions.
- Keep markup accessible and semantic.
- Validate visual quality in realistic list states (short/long names).

## Don'ts

- Don’t change unrelated list logic.
- Don’t use ad-hoc styles that conflict with shared UI patterns.
- Don’t make author text visually compete with title text.

## Open Questions

- Confirm whether long author names should truncate (single line ellipsis) or wrap to two lines.
