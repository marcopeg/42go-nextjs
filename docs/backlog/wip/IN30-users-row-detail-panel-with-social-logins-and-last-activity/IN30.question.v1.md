# Refinement Questions v1 — Users Row Detail Panel With Social Logins and Last Activity

Answer inline under each question, then return to chat and say `next round`.

## Questions

### 1. Detail panel trigger

Should the detail panel open from the whole row click, from a dedicated visible icon/button in the row, or both?

Answer:

whole row. but keep in mind there is the three vertcial dots button that opens the row's menu. that must still work, and "View user details" should be added as first item to that contextual menu.

### 2. Edit panel relationship

Should the read-only detail panel include an `Edit` action that opens the existing edit panel for the same user, or should edit remain only in the three-dot action menu?

Answer:

yes, add an "edit" that triggers the edit panel, and also a "delete" button (ghost button in red with a confirm alert) that is just a placeholder for not. on confirm do an alert with "User deletion not yet implemented".

### 3. User fields to show

Which `auth.users` fields should be shown in the read-only panel?

Current candidate: `id`, `app_id`, `username`, `name`, `email`, `email_verified`, `image`, `profile`, `consent`, `feature_flags`, `created_at`, `updated_at`.

Answer:

- no need for "app_id"
- the "email_verified" should be represented by a "🚧" if not yet verified, and a "✅" if verified. (use better icons if you can). if verified, hovering (or tapping on mobile) on the thick icon should display a popover with the verification date in readable format.

### 4. Sensitive fields

Should any stored fields be explicitly hidden even from backoffice users? For example: password hash must stay hidden; OAuth tokens must stay hidden. Any other field?

Answer:

yes.

### 5. Social login display

For linked `auth.accounts` rows, should the panel show only provider names, or also non-secret metadata such as `type`, `account_id`, `scope`, `expires_at`, `created_at`, and `updated_at`?

Answer:

for now, just use the name to show a nice icon.

### 6. Credentials login display

If a user has no social account rows but can log in with credentials or magic link/email, should the panel show an explicit login method such as `Credentials`, `Email`, or `No social providers linked`?

Answer:

in that case, if there is a password show an icon for "credentials login", else show an icon for "magic link".

### 7. Last activity ordering

Should last activity be selected by `event_at DESC`, by `created_at DESC`, or by `event_at DESC, created_at DESC` as a tie-breaker?

Answer:

event_at desc

### 8. Last activity fields

Should the last activity section show only event name and date, or also event `data` / `meta` summaries when present?

Answer:

- event date
- event name

### 9. Data loading shape

Should `/api/users` return accounts and latest activity for every listed user in the existing list response, or should the page fetch extra detail only after a row is opened?

Answer:

in the list response
