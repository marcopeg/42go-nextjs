# Execution Notes — Users List Social Logins and Last Activity
**Task**: ./IN30.task.md
**Plan**: ./IN30.plan.md

## Decisions

- Use the shared `Modal` component for the read-only user detail panel, matching the existing edit panel pattern.
- Return `hasPassword` as a boolean only; never return the `auth.users.password` hash.
- Return provider names only for social logins; no account IDs, scopes, tokens, or session fields.
- Use text badges for GitHub and Google because the installed `lucide-react` package does not export a GitHub brand icon.
- Use `DISTINCT ON (user_id)` with `event_at DESC, created_at DESC, id DESC` for latest activity.

## Problems Encountered

- Initial `npm run qa` failed because `lucide-react` does not export `Github`. Replaced the icon import with a compact `GH` text badge and reran QA successfully.
- Visual smoke verification reached `https://lc42go.ngrok.app/users` after logging in as `john` / `john`, but the route was blocked by the required LingoCafe profile onboarding overlay.
- A read-only local PostgreSQL validation query initially failed in the sandbox with `EPERM`; rerunning with approved local DB access succeeded.
