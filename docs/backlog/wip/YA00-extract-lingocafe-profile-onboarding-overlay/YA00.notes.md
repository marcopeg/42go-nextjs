## Decisions

- Treat the operator's `execute it` as plan acceptance plus execution.
- Transitioned `YA00` from refined to planned to wip before source edits.
- Implemented the guard config with a `loader` option so LingoCafe can lazy-load the onboarding guard from `AppLayout`.
- Built `LingocafeOnboardingGuard` as the first profile layout guard. It uses `useProfileController`, existing `/api/profile`, shared `ProfileConsent`, and dispatches `profile:complete` after a complete save.
- Removed the `/books` inline onboarding form. `/books` now renders a plain incomplete-profile note and reloads book data on `profile:complete`.
- Adjusted the onboarding Modal to follow the `42go-ui-modal` full-screen overlay page recipe: `presentation="panel"`, `anchor="right"`, `size="full"`, and desktop width forced to the full viewport so no underlying layout strip remains visible.
- Removed the visible Modal header from the onboarding overlay and moved the title/copy into centered page content.
- Switched the onboarding Modal to `anchor="top"` with full desktop height so the panel exits upward when the profile becomes complete.
- Added shared Modal `skipOpenAnimation` and `skipCloseAnimation` props. LingoCafe onboarding uses `skipOpenAnimation` so it appears immediately as a blocker while preserving the upward close animation.

## Problems Encountered

- First `npm run lint` failed on React 19 lint rules: creating a lazy component during render and calling a state-setting loader synchronously inside an effect. Replaced render-time `lazy()` with effect-loaded component state and deferred `/books` initial load with `setTimeout`.
- First `npm run qa` failed because the new profile guard types were not exported from `src/42go/components/ProfileBlock/index.ts`. Exported them and reran QA successfully.
