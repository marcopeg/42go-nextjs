# Manual Test - RO05 App Icons

## Preconditions

1. Start the app with the supported project command, usually `make app.start`.
2. Open browser devtools before loading pages.
3. Keep the Elements, Network, Console, and Application tabs available.
4. Test at least one app with a custom iconset and one app that falls back to `_default`.

## Source Checks

1. Run `rg "logo\\?|public\\.pwa\\.icons|appleTouch180|manifest192|manifest512|maskable512" src`.
2. Confirm live source no longer uses legacy `logo` or `public.pwa.icons` icon paths.
3. Confirm `public/app-icons/_default/` contains every required default asset.
4. Confirm `public/app-icons/lingocafe/` contains the migrated LingoCafe icon assets.

## Browser Favicon Check

1. Open the default app.
2. Inspect the document head.
3. Confirm favicon links point to `/app-icons/_default/...`.
4. Open the LingoCafe host.
5. Inspect the document head.
6. Confirm favicon links point to `/app-icons/lingocafe/...` where LingoCafe provides assets.
7. Confirm the browser Network tab has no failed icon requests.
8. Confirm `/favicon.ico` is not the only favicon mechanism for app-specific pages.

## Apple Touch Icon Check

1. Open the default app.
2. Inspect the document head.
3. Confirm `link rel="apple-touch-icon"` points to `/app-icons/_default/apple-touch-icon-180x180.png`.
4. Open the LingoCafe host.
5. Confirm `link rel="apple-touch-icon"` points to `/app-icons/lingocafe/apple-touch-icon-180x180.png`.

## Manifest Check

1. Request the manifest for the default app.
2. Confirm manifest icons include 192, 512, and maskable 512 entries from `/app-icons/_default`.
3. Request the manifest for the LingoCafe host.
4. Confirm manifest icons include LingoCafe 192 and 512 entries.
5. Confirm maskable 512 falls back to `_default` if LingoCafe has no dedicated maskable file.
6. Confirm `start_url`, `scope`, `display`, colors, name, and description still come from `public.pwa`.

## UI Icon Check

1. Open a public layout page that renders `AppTitle`.
2. Confirm the UI icon uses the resolved app icon.
3. Open a protected app layout page that renders the sidebar/app title.
4. Confirm the protected UI icon matches the resolved app icon.
5. For an app with `public.toolbar.icon`, confirm the toolbar icon overrides the resolved UI icon only for that title surface.

## Fallback Check

1. Temporarily use or create an app without app-specific files under `public/app-icons/<app-id>/`.
2. Open that app.
3. Confirm favicon, Apple touch, manifest, maskable, and UI icon paths fall back to `/app-icons/_default`.
4. Confirm the Console and Network tabs show no missing icon request.

## Strict Validation Check

1. Enable strict validation for one app or set `APP_ICONS_STRICT=1`.
2. Temporarily remove one required app-specific icon asset.
3. Run the build.
4. Confirm the build fails with a clear message naming the missing asset.
5. Restore the asset.
6. Run `npm run qa` and confirm it passes.
