# Migration - RO05 App Icons

## Goal

Move all app identity icon assets and config from legacy scattered fields into the new shared `app-icons` convention.

## Legacy Sources

Current icon-related sources include:

```text
config.logo
config.public.pwa.icons.appleTouch180
config.public.pwa.icons.manifest192
config.public.pwa.icons.manifest512
config.public.pwa.icons.maskable512
public/lingocafe/
public/lingocafe/favicon/
src/app/favicon.ico
```

The root `src/app/favicon.ico` can remain only as the boilerplate browser fallback. It must not be the per-app favicon solution.

## New Asset Layout

```text
public/app-icons/_default/
  favicon.ico
  favicon-16x16.png
  favicon-32x32.png
  apple-touch-icon-180x180.png
  manifest-192x192.png
  manifest-512x512.png
  maskable-512x512.png
  ui.png

public/app-icons/<app-id>/
  favicon.ico
  favicon-16x16.png
  favicon-32x32.png
  apple-touch-icon-180x180.png
  manifest-192x192.png
  manifest-512x512.png
  maskable-512x512.png
  ui.png
```

## New Config Shape

Use `icons` for optional overrides and strictness:

```ts
icons: {
  basePath?: "/app-icons/custom-id";
  strict?: boolean;
  favicon?: {
    ico?: string;
    png16?: string;
    png32?: string;
  };
  appleTouch180?: string;
  manifest192?: string;
  manifest512?: string;
  maskable512?: string;
  ui?: string | ComponentType<{ className?: string }>;
}
```

Most apps should not need explicit paths. The default should be convention-only:

```ts
icons: {
  strict: true,
}
```

## Config Migration

1. Remove `logo` from `TAppConfigItem` and app configs.
2. Remove `icons` from `TPWAConfig`; keep PWA fields for name, display, start URL, scope, colors, and status bar style.
3. Add `icons?: TAppIconsConfig` to `TAppConfigItem`.
4. Add or update app configs to use `icons` only when they need strict mode or explicit override paths.
5. Keep `public.toolbar.icon` as the highest-priority visual title override.

## LingoCafe Asset Mapping

```text
public/lingocafe/favicon/favicon.ico
  -> public/app-icons/lingocafe/favicon.ico

public/lingocafe/favicon/favicon-16x16.png
  -> public/app-icons/lingocafe/favicon-16x16.png

public/lingocafe/favicon/favicon-32x32.png
  -> public/app-icons/lingocafe/favicon-32x32.png

public/lingocafe/favicon/favicon-180x180.png
  -> public/app-icons/lingocafe/apple-touch-icon-180x180.png

public/lingocafe/favicon/favicon-192x192.png
  -> public/app-icons/lingocafe/manifest-192x192.png

public/lingocafe/favicon/favicon-512x512.png
  -> public/app-icons/lingocafe/manifest-512x512.png

public/lingocafe/logo.png
  -> public/app-icons/lingocafe/ui.png
```

If no dedicated LingoCafe maskable asset exists, omit `public/app-icons/lingocafe/maskable-512x512.png` and let the resolver fall back to `_default/maskable-512x512.png` unless LingoCafe is marked strict.

## Documentation Migration

1. Update `contents/default/docs/pwa/README.md` so `public.pwa` no longer documents icon paths.
2. Add a dedicated app icon docs page under `contents/default/docs`.
3. Link the app icon docs page from `contents/default/docs/SIDEBAR.md`.
4. Update AppConfig docs/articles that mention `logo`, `public.pwa.icons`, favicon behavior, or PWA icon paths.
5. Run `rg "logo\\?|public\\.pwa\\.icons|appleTouch180|manifest192|manifest512|maskable512|/lingocafe/favicon|/images/icons" src docs contents` and migrate live references.

## No Compatibility Layer

Do not keep `logo` or `public.pwa.icons` as aliases. This project is early. The old paths should die standing up.
