# Execution Notes — Fix Reader Page Navigation Bounceback
**Task**: ./CX15.task.md
**Plan**: ./CX15.plan.md

## Decisions

- Treat an explicit reader page-turn as authoritative over stale URL-derived route sync from a resumed PWA/browser state.
- Update browser history immediately when the user requests a reader page turn. The visible old page can remain mounted during delayed loading, but the URL and latest route intent move together.
- Guard reader page fetch completion with a latest-route/load-sequence check so older fetches cannot update `bookPage`, `loading`, `error`, or history after a newer intent wins.
- Bind scroll/progress POSTs to the loaded `bookPage` route, not the pending `readerRoute`, so an old mounted page cannot save progress to the new page while a transition is in flight.
- Ignore reader previous/next clicks and mobile swipe page-turns while a page-turn fetch is pending.
