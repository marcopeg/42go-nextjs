# Execution Notes — Promote Events to Core App-Scoped Event System
**Task**: ./WS58.task.md
**Plan**: ./WS58.plan.md

## Decisions

- Execution accepted one core parent table `events.events`, monthly range partitioning by `created_at`, mandatory `app_id`, and lean extraction indexes.
- Historical `lingocafe.events` rows are not migrated in WS58.
- Production cleanup of the old LingoCafe event table remains manual.
- Client event collection is off by default and enabled per app. LingoCafe enables `api:events` with session-required collection.
- Logout recording is best-effort client-side.
- LingoCafe event names do not repeat the app ID. The row is already scoped by `app_id = "lingocafe"`.

## Problems Encountered

- The first backup light smoke still dumped stale `lingocafe.events` rows because the local database still has the old table. The backup script now excludes obsolete `lingocafe.events*` tables in all modes and excludes core `events.events*` in light mode.
- The local Python environment lacks `psycopg`, `pyarrow`, and `duckdb`, so the renamed export skill was validated by syntax/help/no-data paths, not a live export.
- `npx knex seed:run` failed because `knex/seeds/20260427151000.lingocafe.data.js` still deleted from obsolete `lingocafe.events`. The seed now deletes LingoCafe rows from core `events.events` instead, and `npx knex seed:run` passes.
- A browser profile save returned `Event data and meta must be JSON objects.` because event validation was too strict for JSON-like objects returned through the database layer. Validation now accepts plain JSON object tags and normalizes objects through JSON serialization before writing.
- Credentials login events could miss the LingoCafe app scope when the auth callback could not resolve the request app directly. Login/signup recording now prefers `user.appId`, then the resolved request app, then `default`.

## Deviations From Plan

- Step 6 did not need a dedicated script. Migration, direct PostgreSQL smoke, backup smoke, and `npm run qa` covered the needed proof.
- `src/config/lingocafe/profile-hooks.ts` was deleted instead of modified because core profile/consent lifecycle logging replaced the old app-specific hook.

## Additional Requests

- After the user reported no LingoCafe login log and shared the profile-save screenshot, the auth event app-id fallback and JSON object validation were tightened.
- User feedback renamed `book-info`, `page-open`, and `page-scroll` to `book.info`, `page.open`, and `page.scroll`; removed `lingocafe.*` prefixes; and added `page.translate` logging with `cache_type` for both server and browser-cache hits.
- Added reader settings popup events: `read.settings.opened` on popup open and `read.settings.changed` on update/reset.
- Documented events export env vars in `.env.example` and made the exporter fall back from `EVENTS_DATABASE_URL` to `DATABASE_URL`.
- Cursor/partitioning brainstorming was recorded in output: use server ingestion ordering for export safety, keep PostgreSQL hot-cache partitioning on `created_at`, and use `event_at` for Parquet analytics layout after download.

## Known Limitations

- Historical rows in `lingocafe.events` remain where they are.
- A smoke row named `system.smoke` was inserted into the local `events.events` table during validation, then the LingoCafe data seed cleanup removed LingoCafe core event rows during `npx knex seed:run`.
- Full browser auth/profile/logout event proof was not run in Playwright.
