# Execution Notes — Add Image ContentBlock
**Task**: ./TR02.task.md
**Plan**: ./TR02.plan.md

## Decisions

- Started execution after plan acceptance on 2026-05-20.
- Confirmed `image` must be server ContentBlock only. `ContentBlock/client.tsx` stays unchanged.
- Confirmed side content should follow `MarkdownBlock` naming: `source` for inline markdown, `path` for file-backed markdown.
- Confirmed animations are limited to current `ScrollAnimation` values: `fade`, `scale`, `slideUp`; default is `none`.
- Chose `CONTENT_IMAGE_REMOTE_PATTERNS` as the generic Next.js remote image allowlist name. `LC_ASSETS_BASE_PATH` remains only for LingoCafe asset URL generation.
- Added the remote default-page image example conditionally: it renders when `CONTENT_IMAGE_REMOTE_PATTERNS` includes `images.unsplash.com`, preventing unconfigured builds from failing on Next.js remote image validation.
- `npm run qa` passes after implementation.
- HTTP smoke checks passed for `/`, `/docs/composable-pages/image-block`, and `/images/content-blocks/demo-image.svg`; default-app content inspection used `Host: 42go.ngrok.app` because plain localhost did not resolve the default app.

## Problems Encountered

- `npm run qa` first failed because JSX was constructed inside a `try/catch` in `ImageBlock`; moved JSX construction outside the catch. It then failed once on a `string | undefined` path narrowing issue; added an explicit `contentPath` guard.
- Browser automation was not available in this session. The Browser MCP tool did not appear, and the Node REPL environment had no `playwright` module installed.
- Localhost default-app smoke testing needs the app-matching host/headers. A plain `localhost` request can resolve no app and show the development missing-page warning.

## Deviations From Plan

- Visual smoke was limited to `npm run qa`, HTTP checks, local asset checks, and rendered HTML inspection. No screenshot was captured because browser automation was unavailable.

## Additional Requests

- Added four more default home page examples after the initial implementation request: two `left` layouts and two `right` layouts using varied `fade`, `scale`, `slideUp`, and `none` animation combinations.
- `npm run qa` passes after the additional examples. Rendered HTML smoke confirmed all four new headings on the default app home page.
- Added follow-up `valign` support for both `image.valign` and `content.valign` with `top`, `center`, and `bottom`, scoped to left/right layouts. Also fixed animated left/right ordering by moving order/alignment classes onto the animation wrapper.
- Added follow-up `image.unoptimized` support. Loopback URLs such as `http://localhost:4000/...` now default to unoptimized, the default Unsplash demo opts out of optimization explicitly, and development CSP allows direct `http:` image loads.
- `npm run qa` passes after the image optimization follow-up. The first sandboxed run hit a Turbopack internal port permission failure; the same command passed outside the sandbox.
- Added follow-up `image.style` support with `default` and `transparent`. `transparent` removes the image frame border, clipping, and muted background for transparent images.
- Added shared `margin.top` / `margin.bottom` support in the ContentBlock renderer. Values accept Tailwind spacing units directly or responsive dictionaries with `base`, `sm`, `md`, `lg`, `xl`, and `2xl` keys.
- Moved ImageBlock animations from top-level `imageAnimation` / `contentAnimation` to `image.animation` and `content.animation`, and updated default and LingoCafe examples.
- Added native block `padding.top` / `padding.bottom` support. Margin remains renderer-owned, while padding is consumed inside native blocks. LingoCafe hero and first image now set margin and padding to zero.
- Set server page-builder spacing defaults to `margin.top: "[8rem]"`, `margin.bottom: "0"`, and zero internal padding.
- Renamed ImageBlock placement from `image.position` to `image.align` across the implementation, examples, public docs, and agent reference.

## Known Limitations

- The external default-page image example is conditional on `CONTENT_IMAGE_REMOTE_PATTERNS` containing `images.unsplash.com`.
