### 1. Choose the final image sizing API.

For AppConfig string URLs, Next.js cannot automatically infer dimensions at build time. It can infer dimensions only for statically imported local image modules in code, which does not fit this config-driven page API.

`sizes` is the responsive hint for `next/image`: it tells the browser how wide the rendered image will be at different viewport sizes, so the browser can pick the right generated `srcset` candidate.

Which API should this task implement?

Option A, strict and stable:

```ts
image: {
  src: string;
  alt: string;
  width: number;
  height: number;
  sizes?: string;
  position?: "left" | "right" | "top" | "bottom";
}
```

Option B, explicit frame mode:

```ts
image: {
  src: string;
  alt: string;
  width?: number;
  height?: number;
  aspectRatio?: `${number}/${number}`;
  sizes?: string;
  position?: "left" | "right" | "top" | "bottom";
}
```

With option B, callers provide either `width` + `height`, or `aspectRatio`; the component uses a fixed frame and `Image fill`.

Answer:

Go for option A.
Make sure you document the "size" thoroughly and use it in the default config examples.
