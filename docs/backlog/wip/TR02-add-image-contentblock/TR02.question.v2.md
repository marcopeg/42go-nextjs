## Questions

### 1. Confirm the image rendering API.

The strongest fit for developer-authored AppConfig content is probably Next.js `Image`, with explicit dimensions in config:

```ts
image: {
  src: string;
  alt: string;
  width: number;
  height: number;
  position?: "left" | "right" | "top" | "bottom";
  sizes?: string;
}
```

Should this task use that API, or do you want a plain `img` with CSS-controlled sizing?

Answer:

- can the width/height be optional and calculated automatically at build time by nextjs?
- what does "sizes" do?

### 2. Confirm the animation API and defaults.

Since `ScrollAnimation` currently supports only `fade`, `scale`, and `slideUp`, should the image block expose:

```ts
imageAnimation?: "none" | "fade" | "scale" | "slideUp";
contentAnimation?: "none" | "fade" | "scale" | "slideUp";
```

If yes, what should the defaults be when omitted? For example: image defaults to `"slideUp"` and content defaults to `"fade"`.

Answer:

default is "none"
