# Execution Notes — LingoCafe Daily What To Read Next Email
**Task**: ./EQ24.task.md
**Plan**: ./EQ24.plan.md

## Decisions

- Implemented the feature as `42go email lingocafe read-tip`.
- Dry-run is the default; `--send` enables real Resend delivery.
- The command refreshes data with `42go pull all` and `42go query all` unless `--skip-refresh` is passed.
- The default whitelist path is `.local/42go-data/lingocafe_daily_email/whitelist.txt`.
- If the whitelist is missing, it is created with `marco.pegoraro@gmail.com`.
- The sent-email history path is `.local/42go-data/lingocafe_daily_email/sent_emails.parquet`.
- Real send mode reads `LC_RESEND_API_KEY` from the environment or `.env` and never writes it to logs or Parquet.

## Problems Encountered

- The existing codebase uses Resend for LingoCafe magic-link delivery; no Reflow sender exists in the repository.
- The local smoke dry-run created the ignored local whitelist file and selected one whitelisted user, skipping 114 users as not whitelisted.

## Deviations From Plan

- The whitelist file was implemented under `.local/42go-data/lingocafe_daily_email/whitelist.txt` instead of inside a skill folder, because the final implementation target became the `42go` CLI and `.local` keeps pilot recipient state out of git.

## Additional Requests

- User accepted `42go email lingocafe read-tip`.
- User clarified the reminder policy: skip users who read in the last 24 hours and skip users emailed in the last 72 hours.

## Known Limitations

- Real email sending is covered with mocked tests. No live Resend email was sent during execution.
- The whitelist remains a pilot-only gate and should be removed after early testing.
