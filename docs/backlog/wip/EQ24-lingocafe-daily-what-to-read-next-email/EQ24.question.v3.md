## Questions

### 1. Which source should define "user has read in the last 24h"?

Recommended default: use raw `page.open` and `page.scroll` events from `.local/42go-data/events/events_YYYYMM.parquet` when available, because they represent actual reading activity. Use `lingocafe.books_progress.updated_at` as a fallback only if event data is missing for that user/book/page.

Answer:
Use `lingocafe.books_progress` as the first implementation source.

The raw pull already exports this table to `.local/42go-data/lingocafe/books_progress.parquet`, with `user_id`, `book_id`, `page_id`, `progress_bps`, `updated_at`, and `completed_at`. For each user, pick the most recently updated progress row to determine the last read book, current page, scroll position, and whether the user read in the last 24 hours.

Use `books_pages.parquet` to determine page order and whether the current page is the final page of the book. Raw `page.open` and `page.scroll` events are not needed for the first implementation.
