## Questions

### 1. Which email sender should `42go email lingocafe read-tip` use when send mode is enabled?

Recommended default: support a generic HTTP sender configuration first, driven by local environment variables, so you can plug in the provider API you choose without hard-coding it into the recommendation logic. Dry-run remains the default and send mode refuses to run unless sender URL/API key/from address are configured.

Answer:
Use the same existing LingoCafe email delivery service as magic-link login.

The codebase currently wires LingoCafe email auth through the Resend HTTP API using `LC_EMAIL_AUTH_STRATEGY="resend"` and `LC_RESEND_API_KEY` from `.env`, configured in `src/config/lingocafe/config.ts` and sent by `src/42go/auth/lib/email/senders.ts`. The CLI should read `.env`, reuse `LC_RESEND_API_KEY`, and send through the same Resend API endpoint and request shape. Do not copy the secret into docs or logs.
