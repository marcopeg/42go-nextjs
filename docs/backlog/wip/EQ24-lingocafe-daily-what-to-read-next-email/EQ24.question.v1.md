## Questions

### 1. How should the periodic process be invoked in production?

Pick the preferred first implementation shape: a CLI command run by cron/platform scheduler, a protected internal API route called by an external scheduler, a Next.js/server worker pattern, or something else already used in this project.

Answer:
Use a local operator automation for the first version, implemented as a Codex skill or Python script instead of a protected app API route.

The automation should be scheduled by Codex/operator tooling, run `42go pull all`, run `42go query all`, read the local Parquet files, compute eligible recommendations, and write local delivery logs so it can enforce at least 72 hours between sent reminder emails to the same user.

Do not put data-existence discovery inside the future automation flow. During refinement, verify that the needed Parquet files exist and identify anything missing.
