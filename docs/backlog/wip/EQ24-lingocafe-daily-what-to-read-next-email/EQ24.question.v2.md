## Questions

### 1. For the local delivery log, what format and location do you want for the pilot?

Recommended default: write append-only JSONL under `.local/42go-email/lingocafe-daily-reading.jsonl`, with one row per user decision including run id, user id, email hash or email, decision, recommendation type, book/page ids, status, sender message id, error, and timestamps. This is easier to inspect and append than Parquet; query outputs can convert it to Parquet later if needed.

Answer:
Use a Parquet delivery log in the local 42Go data area, alongside the other ignored Parquet files, so it can be inspected with the same local data tooling.

The log should be a full append-style history of sent emails, not just one latest-row table. It should include who received the email, when it was sent, the recommendation/read tip, the plain-text body, the target book/page/link, sender status/message id if available, and enough metadata to enforce cooldown policies.

For now, enforce that at least 72 hours pass between sent reminder emails to the same user.
