# Execution Notes — Stats Users Growth
**Task**: ./BW27.task.md
**Plan**: ./BW27.plan.md

## Decisions

- Preserve `42go events query stats` by converting `stats` into a Typer sub-app whose bare invocation prints the existing archive stats.
- Implement changed-source updates as a bounded recompute per app instead of row-level incremental patching. Unchanged sources read the local aggregate cache; changed sources rebuild app histories from local Parquet. This keeps the result correct while preserving the public cache contract.
- Use DuckDB for cached metric reads to avoid PyArrow native CPU-info warnings leaking into terminal output in the sandbox.

## Problems Encountered

- Initial month/year rolling active metrics used future period ends for partial buckets, which made current month/year WAU and MAU decay incorrectly. Fixed by using the latest available event time for current partial buckets.
- PyArrow emitted sandbox CPU-info warnings when reading cached metric Parquet files. Switched cached reads to DuckDB.

## Deviations From Plan

- Step 11 uses bounded app-history recompute on changed source files rather than true row-level incremental updates. The plan explicitly allowed this fallback when correctness would be stronger.

## Additional Requests

- Operator accepted the plan and requested execution.
- Operator corrected the command path from `42go events query stats users growth` to `42go events query users growth`; implementation and docs were updated accordingly. The old wrong path now fails with no `users` command under `stats`.

## Known Limitations

- Source cache invalidation uses local Parquet path, size, and mtime fingerprints. It is sufficient for the local exporter workflow, but it is not a cryptographic content hash.
