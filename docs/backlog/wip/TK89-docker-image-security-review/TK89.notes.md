# Execution Notes — Docker Image Security Review
**Task**: ./TK89.task.md
**Plan**: ./TK89.plan.md

## Decisions

- Accepted plan moved `TK89` into WIP for execution.
- Review scope remains audit-only: findings and actionable remediation steps go into `TK89.output.md`; Dockerfile/Compose remediation is not performed in this task.
- Docker Hub publication is blocked by the review gate.
- Secret values observed via Compose expansion are intentionally not repeated in the output.

## Problems Encountered

- Initial Trivy image scan failed in the sandbox while downloading the vulnerability database. Retried with escalated permission and the scan succeeded.
- `trivy config --no-progress` failed because `trivy config` does not support `--no-progress` in Trivy 0.70.0. Reran without that flag.
- `docker history` once hit Docker socket permission denial from the sandbox. Reran with escalated Docker daemon access and captured the relevant final-stage history.

## Deviations From Plan

- Runtime standalone behavior was checked with host-specific API smoke tests instead of a full browser/app walkthrough. This was enough to verify proxy execution and app ID resolution for the audit.

## Additional Requests

- Operator clarified that `docker-compose.prod.yml` is a local production environment: it is intended to run the production build locally and mimic production shape for engineers, not define the real production deployment topology.
- Based on that clarification, Compose `env_file`, local DB port exposure, host-gateway, and bind mounts are documented as local-production warnings rather than Docker Hub publishing blockers.
- Operator requested applying the standalone-only runtime image change after a successful trial. The production `Dockerfile` now removes the redundant final full `node_modules` copy.
- Operator bumped the Docker base image from `node:22-alpine` to `node:26-alpine` in all Dockerfile stages to test whether a fresher base clears Docker image vulnerability warnings.
- Operator started a local-production validation with `make prod`; full security-gate runs are intentionally deferred while remediation is in progress.

## Known Limitations

- The review did not remediate any findings.
- The review did not push or inspect a remote Docker Hub image.
- The review did not run a full authenticated application smoke test.
- Node 26 compatibility is pending the operator's local-production `make prod` result.
