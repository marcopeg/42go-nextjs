# Execution Notes — Add 42go Users Pull CLI Export
**Task**: ./LN04.task.md
**Plan**: ./LN04.plan.md

## Decisions

- User/account auth exports will use single Parquet files under `.local/42go-stats/_data/` with sibling JSON state metadata.
- Incremental refresh will use `updated_at` high-water marks and deterministic tie-breakers, then merge changed rows into existing Parquet rows.
- `42go update` calls auth export before event/book/session/users/reads aggregation work and exits on auth export failure.
- The default auth stats root was corrected to `.local/42go-stats` to match the requested local output and the existing analytics stats convention.

## Problems Encountered

- The system Python initially lacked `psycopg`, `pyarrow`, and `duckdb`, so CLI tests could not collect.
- Installed the editable CLI with dev dependencies using `python3 -m pip install -e './cli[dev]'`.

## Deviations From Plan

- Account cursor state includes `provider` after the requested `(updated_at, created_at, app_id, account_id)` tuple to avoid skipping tied account rows when multiple providers share the same account tuple.
