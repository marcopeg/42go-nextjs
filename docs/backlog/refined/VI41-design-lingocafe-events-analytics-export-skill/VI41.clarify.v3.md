# Clarify Round v3 — Design LingoCafe Events Analytics Export Skill

## Questions

### 1. Should each export run create one paired CSV file and one paired Parquet file with the same batch ID?

Recommended layout:

```text
.local/lingocafe-analytics/
  events/
    csv/batch-20260510T163000Z.csv
    parquet/batch-20260510T163000Z.parquet
    state.json
    manifest.jsonl
```

### 2. Which env vars should v1 use?

Recommended:

- `LINGOCAFE_EVENTS_DATABASE_URL` for the source database.
- `LINGOCAFE_ANALYTICS_DIR` for the local store, defaulting to `.local/lingocafe-analytics`.

### 3. Should the cursor advance only after both the CSV and Parquet batch are written and the Parquet file is readable by a smoke query?

## Answers

### 1.

good as you propose

### 2.

LC_EVENTS_DATABASE_URL
(document it in the skill's README.md, and make sure it is easy to change (like a const in the top of the python script in the skill folder))

### 3.

yes, and until an export is completed if the script dies and restart it should be overridden

## Resolution Summary

- Each export run should create one paired CSV file and one paired Parquet file with the same batch ID.
- The accepted default layout is `.local/lingocafe-analytics/events/{csv,parquet,state.json,manifest.jsonl}`.
- The source database env var should be `LC_EVENTS_DATABASE_URL`.
- The source env var name must be documented in the skill README and easy to change through a top-level script constant.
- The cursor should advance only after both outputs are written and the Parquet output passes a smoke read.
- If the script dies before a batch completes, rerunning should overwrite the incomplete batch instead of advancing the cursor or creating duplicates.
