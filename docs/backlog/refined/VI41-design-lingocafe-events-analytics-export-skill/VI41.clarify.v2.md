# Clarify Round v2 — Design LingoCafe Events Analytics Export Skill

## Questions

### 1. Which local storage/query path should the first version use?

Options to decide from:

- Parquet dataset plus DuckDB queries: best fit for columnar analytics, fast scans, compact files, and Python-friendly SQL. More moving parts because writing Parquet needs `pyarrow` or equivalent.
- DuckDB database file: very simple single local artifact and excellent analytics SQL. Less transparent as a filesystem dataset and less append-friendly if the store grows or needs partial file replacement.
- JSONL or CSV batches plus DuckDB read functions: easiest to generate and debug by eye. Larger and slower than Parquet, with more type handling pain for timestamps and JSONB payloads.
- SQLite database: familiar and easy to incrementally upsert. Weaker for analytical scans and nested JSON payloads than DuckDB or Parquet.

## Answers

### 1.

The local analytics store should be treated as the durable event archive. PostgreSQL events are a transitory hot buffer on the VPS, but this task must only download data and must not drop, delete, truncate, or otherwise clean up source events.

The preferred direction is two parallel local stores produced from the same export batch:

- CSV batches for human-readable inspection.
- Parquet batches for Python/DuckDB analysis.

Both stores should be immutable batch outputs. A cursor/manifest should track what was safely exported. JSONL is not needed for v1 unless a later implementation finds a clear reason.

## Resolution Summary

- PostgreSQL should be considered the temporary source buffer, not the long-term event source of truth.
- The Mac-local filesystem store should become the durable analytics archive.
- The export task is read-only against PostgreSQL and must not implement source deletion.
- The v1 archive should write both CSV batches and Parquet batches from each export run.
- CSV exists for human-readable access.
- Parquet exists for Python/DuckDB analysis.
- JSONL is out of scope for v1 unless implementation uncovers a strong need.
