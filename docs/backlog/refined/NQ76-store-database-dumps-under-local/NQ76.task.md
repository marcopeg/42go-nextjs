# Store Database Dumps Under .local

## Elevator's Pitch

Move generated PostgreSQL data-only dump files from `knex/dumps` into a repo-local `.local` location so dump artifacts live with other machine-local operational data instead of inside the Knex migration area.

## Business Gain

Database dumps are local operational artifacts, not migration assets. Keeping them under `.local` reduces confusion around what Knex owns, keeps generated data beside other ignored local archives, and makes backup storage easier to reason about.

## Current State

The `42go-backup` skill writes dumps to `knex/dumps/{utc-timestamp}.dump.{mode}.sql`. `make restore from=<bare filename>` resolves bare filenames from `knex/dumps/`. The skill docs, script constant `DUMPS_DIR`, Makefile-facing examples, and ignore rules all reflect that location.

## Desired State

Generated dump files are written under `.local/42go-backups/`, while preserving current dump filename shape, backup modes, restore behavior, and data-only SQL semantics. Bare restore filenames should resolve only from `.local/42go-backups/`. Explicit readable paths should still work, including old `knex/dumps/<file>.sql` paths when the operator supplies the path directly. Documentation, `.gitignore`, and `.dockerignore` should describe or protect the new location.

## Definition of Success

Running `make backup mode=light` or `make backup mode=full` creates dump SQL under `.local/42go-backups/` and no longer depends on `knex/dumps` as the generated output home. Restore still works from explicit readable paths and from bare dump filenames stored in `.local/42go-backups/`.

- Refinement v1 decisions: use `.local/42go-backups/`; do not search legacy `knex/dumps` for bare restore filenames; remove `knex/dumps/.gitkeep`; update `.gitignore` and `.dockerignore` appropriately.

## Assumptions

- Existing dumps under `knex/dumps` may remain readable only when supplied as explicit paths.
- No migration or schema behavior should change.

## Constraints

- Do not move Knex migrations or seeds.
- Do not change the SQL dump format.
- Do not remove current backup modes.
- Do not run migrations automatically during restore.
- Generated dumps must remain ignored by Git.
- Generated dumps should not be copied into Docker build contexts.
- Do not keep `knex/dumps/.gitkeep` as a primary dump-directory marker.

## Acceptance Criteria

- `make backup mode=light` creates a dump under `.local/42go-backups/`.
- `make backup mode=full` creates a dump under `.local/42go-backups/`.
- `make restore from=<bare dump filename>` resolves from `.local/42go-backups/`.
- `make restore from=<bare dump filename>` does not search legacy `knex/dumps`.
- `make restore from=<explicit path>` still works for readable explicit paths.
- `42go-backup` skill documentation and examples use the new `.local` dump location.
- `.gitignore` covers generated `.local/42go-backups/` dump files and no longer treats `knex/dumps` as the primary generated dump location.
- `.dockerignore` excludes generated local backup artifacts from Docker build contexts.
- `knex/dumps/.gitkeep` is removed.
- Validation includes script syntax/help checks and at least one generated light dump.

## Dos

- Keep the Makefile command surface unchanged: `make backup mode=...` and `make restore from=...`.
- Prefer a single constant/helper for the default dump directory.
- Update error messages that mention the fallback dump path.
- Preserve explicit readable path restore behavior.

## Don'ts

- Do not store generated dumps in tracked folders.
- Do not make operators pass the dump directory for the common case.
- Do not weaken restore SQL error handling.
- Do not touch event export archive behavior in this task.
