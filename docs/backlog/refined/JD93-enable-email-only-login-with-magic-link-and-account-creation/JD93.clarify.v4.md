# Clarify Round v4 — Enable Email-Only Login With Magic Link and Account Creation

## Questions

### 1. For users created by email-only login, should `username` be null, generated from the email, or generated as a random stable handle?

## Answers

### 1.

Default username generation should use the email name portion before `@` when available. If that username is already taken for the app, fall back to the full email address, which must already be unique.

Important: when magic link is enabled together with social logins, different auth providers using the same email must resolve to the same app-scoped user.

## Resolution Summary

- New email-only users should default `username` to the email local-part when available.
- If the local-part username is unavailable, fallback username should be the full email address.
- Email-only login and social login must converge on the same app-scoped user when the verified email matches.
