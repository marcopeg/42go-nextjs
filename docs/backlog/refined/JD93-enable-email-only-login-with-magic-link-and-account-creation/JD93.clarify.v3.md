# Clarify Round v3 — Enable Email-Only Login With Magic Link and Account Creation

## Questions

### 1. Which email sender should be the default target for v1: SMTP/Nodemailer, Resend HTTP API, SendGrid HTTP API, or a custom sender interface?

## Answers

### 1.

This must be a strategy-based implementation so multiple providers can handle the mechanical sending of the email. Different 42Go apps must be able to configure different sender strategies from the available set.

Pick the easiest and best-documented strategy as the first/default implementation.

## Resolution Summary

- Email sending must use a strategy/provider abstraction.
- Sender strategy must be configurable per 42Go app.
- First/default strategy should be selected by implementation pragmatism and documentation quality.
- Proposed default for planning: SMTP/Nodemailer, because it aligns with the current NextAuth v4 EmailProvider path and is documented by NextAuth/Auth.js.
