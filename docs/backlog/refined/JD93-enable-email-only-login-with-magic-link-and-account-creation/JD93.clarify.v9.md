# Clarify Round v9 — Enable Email-Only Login With Magic Link and Account Creation

## Questions

### 1. For local development, should email delivery default to console output, an SMTP sink, provider sandbox mode, or a mocked sender?

## Answers

### 1.

Use a different approach: create a real `console` email sender strategy and select it by default.

Auth provider config should allow multiple sender strategies as key/value entries, plus a `useStrategy` key defaulting to `console`. The selected strategy property must be configurable via environment variable.

## Resolution Summary

- Local/default email delivery uses a `console` sender strategy.
- `console` is a first-class sender strategy, not a special local-only mock.
- Email provider config supports multiple named sender strategies as key/value entries.
- Provider config includes a selected strategy key, tentatively `useStrategy`, defaulting to `console`.
- The selected sender strategy must be configurable from an environment variable.
