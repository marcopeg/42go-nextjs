# Clarify Round v17 — Enable Email-Only Login With Magic Link and Account Creation

## Questions

### 1. Should email provider config be split into `public` UI metadata and server-only `server` secrets/settings?

## Answers

### 1.

Use the same simple approach as the current social providers for v1.

The current configuration already puts OAuth provider client IDs/secrets in the `auth.providers` config using environment variables. Email provider config should follow that pattern for now. Do not introduce a full public/server AppConfig split in this task. Iterate and harden later if needed.

## Resolution Summary

- V1 does not introduce a public/server AppConfig split.
- Email provider config follows the existing social-provider pattern under `auth.providers`.
- Secrets/settings may be referenced from environment variables in the provider config as current OAuth providers do.
- A future hardening task can split public provider metadata from server-only secret config.
