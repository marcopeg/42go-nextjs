# Clarify Round v1 — Enable Email-Only Login With Magic Link and Account Creation

## Questions

### 1. Should v1 require manual code entry, or can v1 ship as magic-link-first while we evaluate code entry as a follow-up?

## Answers

### 1.

By default it should do both. After the user enters the email, the flow should show an "enter the code" screen and also provide a resend-email action.

The auth strategy configuration should remain open for custom behavior later. It should support throttling resend/request attempts, with a default delay of 1 minute. Prefer support for a progressive delay list such as `15s,30s,1m,3m,7m,15m`, where each subsequent resend is delayed accordingly and the UI shows a customizable error message.

## Resolution Summary

- v1 must support both magic-link click and manual code entry by default.
- The post-email-request screen should focus on code entry and expose resend.
- Provider configuration should include throttling, with a default 1-minute delay and optional progressive resend delays plus configurable messaging.
