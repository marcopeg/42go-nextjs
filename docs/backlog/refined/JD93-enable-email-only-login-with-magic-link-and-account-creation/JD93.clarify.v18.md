# Clarify Round v18 — Enable Email-Only Login With Magic Link and Account Creation

## Questions

### 1. Should the completion gate require a validation matrix covering credentials-only, email-only, credentials+email, new user, existing user, expired/reused code, throttled resend, console sender, and Resend sender?

## Answers

### 1.

Yes.

Before the task is considered done, the implementation must prove the critical auth cases work: app with only password login, app with only email-code login, app with both password and email-code login, new user created by email, existing user logs in by email, expired/already-used code fails, resend is throttled, console sender works, and Resend sender works.

## Resolution Summary

- Completion requires a validation matrix for credentials-only, email-only, and credentials-plus-email app configurations.
- Completion requires validation for new user creation, existing user login, expired/reused code failure, resend throttling, console sender, and Resend sender.
