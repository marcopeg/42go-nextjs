# Clarify Round v15 — Enable Email-Only Login With Magic Link and Account Creation

## Questions

### 1. For manual code entry, should we implement a dedicated `/api/auth/email/verify-code` route that consumes the same stored verification token and then hands off to NextAuth session creation?

## Answers

### 1.

Yes, but manual code verification must use the same logic as magic-link verification so there is no duplication after the verification step.

## Resolution Summary

- V1 should include a dedicated manual-code verification route.
- Manual code verification must consume the same stored verification-token model as magic links.
- After token/code verification succeeds, manual code and magic-link flows must converge on the same user/session creation logic.
- Do not duplicate account creation, token consumption, or NextAuth session creation behavior in two separate paths.
