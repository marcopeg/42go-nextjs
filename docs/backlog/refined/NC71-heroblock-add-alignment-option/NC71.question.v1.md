# Refinement Questions v1 — HeroBlock: Add Alignment Option

Answer inline under each question, then return to chat and say `next round`.

## Questions

### 1. Which HeroBlock fields must `alignment` affect?

Should `alignment: "left" | "center" | "right"` control only text alignment for `title` and `subtitle`, or should it also control the action button row justification?

Answer:

It should control all elements. 

### 2. What should the default and backward compatibility behavior be?

The draft says the option is optional and defaults to `"center"`. Confirm whether every existing HeroBlock with no `alignment` must render exactly as it does today.

Answer:

The default is "center" with is the current behavior.

### 3. How should alignment interact with width and spacing?

Today title and subtitle use centered max-width classes such as `mx-auto`. For left/right alignment, should the content remain in the same centered max-width column, or should the column itself move left/right within the hero container?

Answer:

if it's aligned left, i expect all the elements to start at the same distance from the left vieport border.
the opposite for right align.

### 4. How should alignment behave with `backgroundImage`?

Should left/right alignment work the same when a hero has a background image and overlay, or does background-image mode need separate padding, contrast, or positioning rules?

Answer:

so far, use it to align the starting origin of the background image.

there should be a heroblock documentation in the /content folder. if not, add it and document it.
