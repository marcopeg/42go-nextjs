# Refinement Questions v1 - Add 42go Backup CLI Commands

Answer inline under each question, then return to chat and say `next round`.

## Questions

### 1. Should the CLI surface be strictly positional?

Proposed default:

- `42go backup full`
- `42go backup light`

No `42go backup --mode full` alias unless there is a strong reason. Fewer doors. Fewer bruises.

Answer:

i'd say via flag. like "--light" or "--full". if not specified, i'd expect an inline multiple choice menu that makes the user choose which mode to run

### 2. Should `make backup` use the same local CLI fallback style as `make events`?

Proposed default:

- If `42go` exists on PATH, run `42go backup "$(mode)"`.
- Else if a repo-local CLI executable exists, run that.
- Else print an install hint and fail.

This keeps `make backup mode=full` working for operators who have not installed the CLI globally.

Answer:

assume `42go` is always available and fail hard if it is not. all the logic should be moved to the cli.

### 3. Is restore intentionally out of scope for this task?

Proposed default:

- Add only `42go backup full|light`.
- Keep `make restore from=...` on the existing backup script for now.
- Do not add `42go restore` unless a later task asks for it.

Answer:

add also the `42go restore --from xxx` if the "--from" is not set, show the list of availble backups sorted by date from newer to older so that it's easy to choose.

move the whole logic to the cli and keep the make as wrapper interface
